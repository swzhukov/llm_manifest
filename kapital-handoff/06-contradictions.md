## 🔄 Конфликты в текущей архитектуре (мои сомнения)

### A1. State в JSON vs Postgres

Сейчас state в `/opt/beget/kapital/data/state.json`. На 1 user ОК. На 10+ users — race conditions, нет миграций. Стоит ли мигрировать в Postgres (используется для n8n data)?

### A2. n8n как backbone vs Flask

Сейчас 127 нод в n8n. Это неподдерживаемо. Альтернатива:
- Всё в Flask (просто, но PM теряет визуальный workflow editor)
- Всё в n8n (текущее, но растёт)
- Гибрид: только routing в n8n, вся логика в Flask (что я начал делать в Sprint 4)

### A3. Polling vs Webhook

Webhook работает, polling сломан. Это нормально? РФ блокирует webhook inbound, **поэтому polling критичен** для других пользователей. Но PM = единственный user, и он использует webhook.

### A4. Bot token в plain text vs env var

Все credentials (bot token, Yandex API key, folder ID) в n8n workflow JSON в plain text. Секреты текут через экспортированный JSON. Нужно вынести в env vars.
