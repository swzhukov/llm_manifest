# Decision tree: обновлять ли wiki? (wiki-curator)

Используй это **дословно** в своём внутреннем рассуждении перед тем как что-то менять в `swzhukov/llm_manifest`.

## CoT (рассуждение вслух)

```
После значимой задачи пройди это дерево ПОСЛЕДОВАТЕЛЬНО.
Не пропускай шаги. Не угадывай — отвечай на каждый вопрос.
```

## Дерево

### Q1. Появилась ли НОВАЯ ИНФОРМАЦИЯ?
*(т.е. то, чего нет в wiki/ENVIRONMENT/MISTAKES)*

- **НЕТ** → обновление не нужно. Сообщи: «wiki не требует обновления». Стоп.
- **ДА** → переходи к Q2.

### Q2. Это общий урок / паттерн / новая среда?
*(применим в будущих проектах, не только этот раз)*

- **НЕТ** → переходи к Q3.
- **ДА** → обновление НУЖНО. Переходи к Q4.

### Q3. Это полезно для будущих сессий?

- **НЕТ** → не обновлять. Записать в локальный scratchpad (memory topic `scratchpad` или комментарий в чате).
- **ДА** → обновление НУЖНО. Переходи к Q4.

### Q4. Это структурное изменение wiki?
*(новый файл, новая папка, изменение SCHEMA.md/AGENTS.md)*

- **ДА** → требует **явного одобрения пользователя**. Не auto-commit. Покажи план.
- **НЕТ** → auto-commit через `scripts/commit-wiki.sh`. Переходи к Q5.

### Q5. Уже есть запись про это в wiki?

- **ДА** → **дополнить**, не дублировать. Используй `git log` чтобы найти старую запись.
- **НЕТ** → создать новую запись в правильном файле (см. таблицу ниже).

## Куда писать

| Тип инфы | Файл | Формат |
|----------|------|--------|
| Новая ошибка с root cause | `MISTAKES.md` | `### X.Y ❌ [название] / Когда / Симптом / Причина / Решение` |
| Изменение/добавление сервиса | `ENVIRONMENT.md` | по схеме существующих секций |
| Новый эффективный паттерн | `wiki/COUNCIL_RECOMMENDATIONS.md` или новый `wiki/topics/<name>.md` | с примером и контр-примером |
| Устаревшее | добавить `status: superseded` в frontmatter | **не удалять** |
| Открытый вопрос на будущее | `wiki/pending.md` | `- [ ] вопрос (из сессии YYYY-MM-DD)` |
| Конец сессии | `wiki/session-summaries.md` | новая секция «Сессия N» |
| **Любое изменение** | `wiki/log.md` | строка `## YYYY-MM-DD (auto-update)` |

## Триггеры «точно нужно обновить»

- 🚨 Новая ошибка, которая повторялась 2+ раза
- 🚨 Новая среда / сервис
- 🚨 Новый эффективный паттерн
- 🚨 Изменение интерфейса существующего сервиса
- 🚨 Конец сессии (обновить `session-summaries.md`)

## Триггеры «точно НЕ нужно обновлять»

- ❌ Решение task-specific проблемы
- ❌ Одноразовый хак
- ❌ Переписывание (если не о wiki)
- ❌ Личные предпочтения пользователя (одноразовые)

## Чего НЕ делать (даже если Q1=ДА)

- ❌ Перезаписывать `wiki/SCHEMA.md` или `wiki/AGENTS.md` без одобрения
- ❌ Дублировать существующие записи
- ❌ Писать секреты в wiki (даже в `.env.example` — MISTAKES §14.8)
- ❌ Удалять файлы (помечай `status: superseded`)
