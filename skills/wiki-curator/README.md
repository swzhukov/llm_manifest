# wiki-curator (Mavis skill)

Анализирует завершённую задачу про автоматизацию и решает, нужно ли обновить базу знаний [swzhukov/llm_manifest](https://github.com/swzhukov/llm_manifest). В режиме `CAN-WRITE-FULL` (есть `GITHUB_LLM_MANIFEST_TOKEN`) — коммитит напрямую. В других режимах — готовит diff для ручного применения.

## Состав

- `SKILL.md` — основная инструкция
- `references/decision-tree.md` — Q1-Q5 дерево решений «обновлять ли вики»
- `scripts/commit-wiki.sh` — safe commit+push с retry

## Когда применяется

В **конце значимой задачи**, если был хотя бы один из:
- Новая ошибка, повторявшаяся 2+ раза
- Новая среда/сервис
- Новый эффективный паттерн
- Изменение интерфейса существующего сервиса
- Конец сессии (обновить `session-summaries.md`)

## Режимы записи

| Режим | Условие | Что делает |
|-------|---------|------------|
| `CAN-WRITE-FULL` | есть `GITHUB_LLM_MANIFEST_TOKEN` | коммитит напрямую через `scripts/commit-wiki.sh` |
| `CAN-READ-ONLY` | нет токена | готовит `git apply` патч для ручного применения |
| `DEGRADED` | репо недоступно | выводит полный текст изменённых файлов |
| `OFFLINE` | нет сети | то же, что DEGRADED + отложить |

## Связь

- Вызывается **после** `wiki-loader` и `gstack-team` (Phase 7 ship)
- **Не** коммитит автоматически изменения в `wiki/SCHEMA.md` и `wiki/AGENTS.md` (требуют одобрения)
