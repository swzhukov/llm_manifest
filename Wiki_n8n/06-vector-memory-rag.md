# 06 — Векторная память и RAG

> Основано на уроках 8 + 9 курса George Erman + обсуждения в чате.

RAG (Retrieval-Augmented Generation) — это когда LLM отвечает на вопросы **не из своей памяти** (где он мог нафантазировать), а **из твоей базы документов**. Векторная память — это способ найти релевантный документ быстро, даже если формулировка вопроса не совпадает с текстом в документе.

## 6.1 Два вида памяти бота

| | Обычная (history) | Векторная (RAG) |
|---|---|---|
| Что хранит | диалог `user → bot → user` | чанки документов + embeddings |
| Зачем | помнить, о чём говорили | отвечать на фактологические вопросы |
| Где живёт | таблица `chat_history` (Postgres) | таблица `documents` + колонка `embedding vector(1536)` |
| Как ищет | по `chat_id` + `created_at` | по **семантической близости** векторов |
| Лимит | сколько угодно (но контекст LLM ограничен) | неограниченно по объёму |
| Аналогия | «история переписки» | «база знаний компании» |

> Из урока 8: «В прошлом занятии мы победили вопрос, как сделать практически бесконечную память у бота. Но кроме памяти, ему же нужно ещё и интеллект — конкретное знание о компании».

## 6.2 Как работает RAG (на пальцах)

```
Шаг 1: INGEST (загрузка документов)
   Документ → split на чанки → embedding → INSERT в documents

Шаг 2: QUERY (поиск ответа)
   Вопрос юзера → embedding → сравнить с embedding всех чанков
   → взять top-K (например, 5) самых похожих
   → передать их в LLM как контекст
   → LLM генерирует ответ на основе контекста
```

Из урока 8: «Когда задаём вопрос ИИ, он превращает наш вопрос в вектор. И идёт сравнение существующих векторов с тем, что получился. И минимальное расхождение будет являться неким ответом».

## 6.3 Чанки: размер и overlap

### Что такое чанк

Кусок текста, на который разбивается документ. Из урока 8: «Чанки — это блоки с текстом, как абзацы. Размер можно регулировать».

### Параметры Text Splitter

| Параметр | Типичное значение | Зачем |
|---|---|---|
| `chunk_size` | 500–1500 символов (или 200–500 tokens) | баланс между точностью и контекстом |
| `chunk_overlap` | 10–20% от chunk_size | чтобы не потерять границы смыслов |

### Почему overlap важен

Из урока 8: «Если ответ на вопрос находится в конце одного чанка и в начале другого — какая-то информация может потеряться. Поэтому делают перехлёст».

Пример:
```
Чанк 1: "...компания основана в 2015 году[END]"
Чанк 2: "[START]Основатели компании — Иванов и Петров..."

С overlap=50:
Чанк 1: "...компания основана в 2015 году. Основатели..."
Чанк 2: "Основатели компании — Иванов и Петров..."
```

✅ **Best practice:** `chunk_size = 1000`, `chunk_overlap = 200` для большинства случаев (из курса).

## 6.4 Embedding-модели

| Модель | Размерность | Цена | Когда |
|---|---|---|---|
| `text-embedding-3-small` (OpenAI) | 1536 | $0.02/1M tokens | по умолчанию в курсе |
| `text-embedding-3-large` (OpenAI) | 3072 | $0.13/1M tokens | если small не хватает точности |
| `text-embedding-ada-002` (OpenAI) | 1536 | $0.10/1M tokens | старая, но работает |
| `embeddings` (YandexGPT) | 256 | дешевле | если уже на Yandex |

⚠️ **Важно:** размерность должна совпадать в `CREATE TABLE documents (embedding vector(1536))` и в ноде Vector Store. Меняешь модель — меняй схему.

## 6.5 Metadata filtering

Когда у тебя в одной таблице документы из разных источников (договоры, FAQ, регламенты), фильтруй по полю `metadata.type`.

Из урока 8: «Мы фильтруем по типу, у которого параметр `info_hub_docs`. Если не проставить — непонятно, каким образом сравнивать документы».

Пример metadata в workflow:
```json
{
  "type": "info_hub_docs",
  "source": "company_website",
  "ingested_at": "2026-01-15"
}
```

В ноде Supabase Vector Store:
```
Metadata Filter: { "type": "info_hub_docs" }
```

## 6.6 Temperature для RAG

Из урока 8: «Чем больше к единице — тем более творческий подход. Нам не нужна творческая мысль, потому что у нас чёткие данные, от которых нельзя отходить. Ставим минимальное значение температуры».

✅ **Best practice:** для RAG-ботов `temperature = 0` или `0.1` (минимум, чтобы ответ был строго по документу).

Для креативных задач (копирайтинг) — наоборот, `temperature = 0.7–1.0`.

## 6.7 Размер контекста и стоимость

RAG передаёт в LLM топ-чанки как контекст. Чем больше чанков — тем больше токенов — тем дороже.

| Top-K | Контекст | Стоимость (GPT-4) | Точность |
|---|---|---|---|
| 1 | 1к токенов | $0.03 | низкая |
| 3 | 3к токенов | $0.09 | средняя |
| 5 | 5к токенов | $0.15 | высокая |
| 10 | 10к токенов | $0.30 | максимум, но может «зашуметь» |

✅ **Best practice:** начинай с **top-3–5**. Если бот не находит ответ — увеличь. Если отвечает невпопад — уменьши `chunk_size`.

## 6.8 Векторная память для диалога (а не для документов)

Можно использовать Vector Store **вместо обычной `chat_history`**: каждое сообщение юзера превращать в вектор, а при новом вопросе — искать top-K похожих прошлых сообщений. Это даёт «семантическую память» — бот помнит, что юзер спрашивал про доставку, даже если новый вопрос звучит иначе.

⚠️ Дороже, чем обычная `chat_history`, но иногда оправдано.

## 6.9 Типичные ошибки

| Симптом | Причина | Решение |
|---|---|---|
| Бот отвечает «не знаю» | top-K слишком мал, или чанки слишком большие | увеличить top-K, уменьшить chunk_size |
| Бот отвечает невпопад | embedding-модель не та, или dimensions не совпадают | проверить `vector(1536)` в схеме |
| Бот фантазирует | temperature высокий | поставить 0 |
| «column embedding does not exist» | не выполнил CREATE EXTENSION vector | см. [05-supabase.md §5.3](05-supabase.md) |
| Чанки не загружаются | Google Drive триггер не сработал, или файл не в той папке | проверить folder ID, права доступа |
| Бот игнорирует часть документов | metadata filter отрезает | проверить значение в `metadata.type` |
| Embedding стоит дорого | большие чанки, много документов | `chunk_size = 500`, `text-embedding-3-small` |

## 6.10 Альтернативы Supabase Vector Store

| Платформа | Стоимость | Плюсы | Минусы |
|---|---|---|---|
| Supabase + pgvector | бесплатно до 500 МБ | self-hosted, контроль | надо настраивать |
| Qdrant | бесплатно self-hosted | быстрый, специализированный | отдельный сервис |
| Pinecone | $0.096/мес за 100K векторов | zero-ops | дорого на больших объёмах |
| Weaviate | бесплатно self-hosted | гибридный поиск | сложнее в настройке |
| Chroma | бесплатно | простой | не для прода |

✅ **Best practice (для пользователей из курса):** Supabase Vector Store — потому что у тебя уже есть Supabase в docker-стеке.
