# 02 — Интерфейс и узлы (ноды)

> Основано на уроках 4, 5, 6 курса George Erman.

## 2.1 Дашборд

При входе в n8n видишь три вкладки:

1. **Personal / Workflows** — список workflow, поиск, сортировка, теги.
2. **Credentials** — все сохранённые доступы (Telegram, OpenAI, Supabase, ...).
3. **Executions** — история запусков. Фильтруй по ошибкам. Автообновление — галочка сверху.

Из урока 4: «В идеале здесь хранятся исключительно только с ошибкой executions. Потому что success-full неинтересно».

## 2.2 Холст (canvas) — управление

| Действие | Шорткат / кнопка |
|---|---|
| Двигать весь холст | средняя кнопка мыши / `Space + drag` / `Ctrl + drag` |
| Zoom | `Ctrl + scroll` |
| Вернуться к центру | кнопка-«щётка» в правом нижнем углу |
| Переименовать workflow | клик по названию сверху |
| Добавить тег | поле «Tags» сверху, Enter |
| Сохранить | `Ctrl+S` или кнопка Save |
| История изменений | кнопка «история» в верхней панели (доступна с разблокированной лицензией) |

> Из урока 4: «История не показывает внутренние изменения в нодах, только структурные: подвинул, удалил, добавил. Но если случайно что-то сломал — откатить можно».

## 2.3 Два типа нод

> Из урока 6: «Узлы могут выполнять всего две операции. Первая — когда узел является триггером. Вторая — когда узел является экшеном».

| | Trigger (триггер) | Action (экшен) |
|---|---|---|
| Вход | ❌ | ✅ |
| Выход | ✅ | ✅ |
| Форма углов | закруглённые | квадратные |
| Где стоит | только в начале цепочки | везде после триггера |
| Что делает | запускает workflow по событию | выполняет действие / обрабатывает данные |

## 2.4 Базовые триггеры (90% задач)

| Триггер | Пример использования |
|---|---|
| **Manual Trigger** | ручной запуск, тестирование |
| **n8n Form Trigger** | «оставьте заявку» — публичная форма |
| **Email Trigger (IMAP)** | «пришло письмо от client@» — автоответ |
| **Schedule Trigger** | «каждый понедельник в 9:00» — рассылка |
| **Telegram Trigger** | «кто-то написал боту» — чат-бот |
| **Webhook** | «пришёл POST на /hook» — внешний сервис дёргает нас |
| **Local File Trigger** | «в папке /dropbox появился файл» — загрузка |

> Из урока 6: «90% того, что вам понадобится. Сверху вниз, слева направо».

## 2.5 Базовые экшены

| Экшен | Назначение |
|---|---|
| **HTTP Request** | вызов любого REST API (GET / POST / PUT / DELETE) |
| **Code (JS / Python)** | произвольный код внутри одной ноды |
| **Set / Edit Fields** | поменять значения полей в потоке данных |
| **IF** | простое ветвление `if X then Y else Z` |
| **Switch** | многоусловный выбор (замена спагетти из IF) |
| **Merge** | склейка двух параллельных веток |
| **Loop / Split In Batches** | обработать массив поштучно |
| **Wait** | подождать N секунд / до определённого времени |
| **Sticky Note** | комментарий на холсте (см. §2.5) |

> Из урока 4: «У каждой ноды есть мини-менюшка... очень важная кнопочка — **деактивировать ноду**. Когда деактивируем, она перестаёт быть активна, сквозь неё проходят данные. Если сбой — не удаляй, просто деактивируй».

## 2.6 Sticky Notes — зачем

Это «стикеры» в холсте. По дефолту находятся **за нодами** — в них можно положить ноды как в контейнер.

Используются для:
- **Документации** прямо внутри workflow
- **Группировки** (например, «этап 1: собрать данные»)
- **Пояснений** для клиента / коллеги

Поддерживают **Markdown**: дважды клик → пишешь текст → он рендерится как Markdown.

Из урока 4: «Шаблоны в интернете все снабжены стикерами, которые описывают, что workflow делает. Можешь два раза кликнуть и в формате Markdown писать всё, что хочешь».

## 2.7 Sticky Note: best practice

- **Один цвет** на этап (например, синий = триггер, зелёный = действие, красный = ошибка).
- **Markdown-заголовок** с кратким описанием шага.
- **Кто менял / когда** — внизу мелким шрифтом.

## 2.8 Три режима ноды на холсте

| Иконка | Что делает |
|---|---|
| ▶ Play | прогнать **только эту ноду** с текущими входными данными |
| ⏸ Deactivate | **выключить** ноду (данные проходят насквозь) |
| ⊕ Execute Workflow | прогнать **весь workflow** от начала |

## 2.9 Дополнительные возможности

- **Экспорт workflow** — три точки → `Download` → JSON. Это твой бэкап.
- **Импорт** — `Import from File` (свой JSON) или `Import from URL` (⚠️ Георгий не рекомендует — вопросы безопасности).
- **Дубликат** — три точки → `Duplicate` (копия с новым именем).
- **Теги** — для фильтрации. Из урока 4: «Теги — это очень удобная вещь, замечательно работает на моменте фильтрации».
- **Папки** (доступны с разблокированной лицензией) — для группировки по клиентам / направлениям.

## 2.10 Кнопка «Execute Workflow» vs активный режим

| | Execute | Активация (зелёный тумблер) |
|---|---|---|
| Что делает | один раз прогоняет workflow | запускает по триггеру (Telegram, Schedule, ...) |
| Когда использовать | отладка, тест | прод |
| Workflow должен быть «завершён»? | не обязательно | да, иначе кнопка неактивна |

> Из урока 4: «Актив/inactive мы не можем запустить, потому что у нас только одна нода. Когда процесс завершён — система себя перепроверяет и, если не находит ошибки, запускается».
