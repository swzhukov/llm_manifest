# 01 — Установка и обновление n8n

> Основано на уроках 1, 2, 3 курса George Erman + живая сессия установки из `040508f4` (Андрей Балакирев ставит с нуля под руководством Георгия, октябрь 2025).

## 1.1 Установка одной командой

Используется **`n8n-installer`** от Юрия Косаковского. Это обёртка, которая разворачивает Docker-стек с Caddy (reverse proxy + auto-SSL) и n8n за один заход.

```bash
# 1. Зайти на сервер по SSH
ssh root@your-server

# 2. Скопировать команду с GitHub
#    https://github.com/skosakivskii/n8n-installer
#    (форк Юрия Косаковского; основной апстрим может отличаться)

# 3. Вставить в Termius: Ctrl+Shift+V, Enter

# 4. Интерактивно выбрать пакеты — по умолчанию ставить ТОЛЬКО n8n,
#    снять все звёздочки кроме n8n
#    (Supabase / Flowise / ComfyUI / Caddy ставить по необходимости)
```

⚠️ **Подводный камень:** если уже стоял n8n, и ты поставил ещё раз с теми же путями — данные **не потеряются** (volume `n8n_data` сохраняется). Но если ставишь в другой домен и другой volume — данные перетрутся. Подробнее §1.4.

## 1.2 Что инсталлер поднимает

```
n8n        — основной сервис
Caddy      — reverse proxy + Let's Encrypt
Postgres   — БД для n8n
Caddy      — SSL
(опционально) Supabase, Flowise, ComfyUI, Crawl4AI, Dify
```

> Из урока 7 курса: «Все пакеты внутри n8n-installer... Supabase ставится рядом с n8n. Не используй команду установки для апгрейда — она стирает данные».

## 1.3 Установка Supabase рядом с n8n

```bash
# 1. Зайти в папку n8n-installer (НЕ корень!)
cd ~/n8n-installer

# 2. Взять команду обновления (НЕ установки) с GitHub
#    https://github.com/skosakivskii/n8n-installer  → README → upgrade command

# 3. Вставить в терминал
bash ./upgrade.sh  # или актуальное имя скрипта

# 4. В интерактивном меню:
#    - n8n   — оставить звёздочку (*)
#    - Supabase — поставить звёздочку (*)
#    - всё остальное (Flowise, ComfyUI) — снять
#    - домен уже введён, Enter
#    - "потоки" — Enter (default 1)

# 5. Дождаться перезапуска контейнеров
# 6. Зайти на n8n.новый-поддомен — должен работать
# 7. Зайти на supabase.поддомен — появится панель
```

✅ **Best practice:** сначала ставим **только n8n**, проверяем, что работает, потом **отдельной командой обновления** добавляем Supabase. Не пытаемся поставить всё сразу.

## 1.4 Обновление с сохранением данных

```bash
# ⚠️ ГЛАВНОЕ ПРАВИЛО:
# Обновление = upgrade-скрипт (тот же, что для добавления Supabase)
# Установка = install-скрипт (СТИРАЕТ ДАННЫЕ)

# Алгоритм:
# 1. cd ~/n8n-installer
# 2. Запустить upgrade-скрипт
# 3. Снять все звёздочки кроме тех сервисов, которые хочешь обновить
# 4. Дождаться перезапуска
# 5. ВСЕ credentials, workflows, executions — сохранятся
```

## 1.5 Где лежат секреты

После установки все доступы в файле:

```
/root/n8n-installer/.env
```

Что там:
```
# n8n
N8N_HOST=n8n.yourdomain.ru
N8N_USER=admin
N8N_PASSWORD=...

# Postgres
POSTGRES_DB=n8n
POSTGRES_USER=n8n
POSTGRES_PASSWORD=...
POSTGRES_HOST=postgres
POSTGRES_PORT=5432

# Supabase
SUPABASE_PASSWORD=...
SERVICE_ROLE_KEY=eyJhbG...  # ← это для Vector Store в n8n

# Домены
N8N_DOMAIN=n8n.yourdomain.ru
SUPABASE_DOMAIN=supabase.yourdomain.ru
FLOWISE_DOMAIN=...
```

✅ **Best practice:** первым делом после установки — открыть `.env`, скопировать в надёжное место (1Password / KeePass / Google Doc). Сервисный ключ Supabase (SERVICE_ROLE_KEY) тебе понадобится для векторной памяти в n8n.

## 1.6 Нюансы Beget

Из живой сессии (`040508f4`):

- **Сервер:** Европа, Латвия (Standard 3+ GHz, 2 vCPU, 2 ГБ RAM, 30 ГБ SSD) — **достаточно для n8n + Supabase + 5–6 workflow**.
- **Стоимость:** ~2 200 ₽/мес.
- **Где смотреть логи установки:** в терминале (Termius / iTerm).
- **Reverse proxy** делает **Caddy** (он внутри стека), отдельно nginx поднимать **не надо**.
- **Домен** должен смотреть на сервер через DNS. Установщик спрашивает «is the primary domain?» — отвечаешь `yes`, перемещаешь стрелкой на нужный домен и Enter.
- **Импорт готовых workflows** при установке: `Import -300 ready-made n8n workflows now? This can take ~30 minutes.` — **пропускай**, чтобы не замусорить и не затянуть установку.
- **Количество n8n-воркеров:** `Enter number of n8n workers to run (default 1)` — для прод-нагрузки ставь **5** (по совету Георгия).

⚠️ **Подводный камень:** Termius вставляет с задержкой, читай подсказки и не торопись. Из чата: «Ааа! Вот теперь понятно, в чём моя ошибка была!» — стандартная фраза, не паникуй.

## 1.7 Бэкап credentials

В `.env` хранятся **внешние** секреты (БД, домены). **Credentials, которые создаёт сам n8n** (API-ключи к внешним сервисам, токены Telegram-ботов) — лежат в Postgres. Бэкап:

```bash
# Вариант 1: через n8n UI
# Settings → Usage → Export credentials

# Вариант 2: бэкап всего Postgres
docker exec -t n8n-postgres pg_dump -U n8n n8n > backup-$(date +%F).sql
```

> Из чата: «Если обновление сожрало .env, проверь, что credentials подтянулись из бэкапа. Бэкап credentials встроен в n8n 2.1+ — он сам подхватит».

## 1.8 Типичные ошибки при установке

| Симптом | Причина | Решение |
|---|---|---|
| Docker не поднимается | не встал Docker на чистый сервер | проверить `docker ps`, перезапустить `systemctl start docker` |
| Домен не открывается | DNS не сделан / долго обновляется | `nslookup yourdomain.ru`; подождать 5–10 мин |
| 500 при первом запуске | не создал аккаунт при первом входе | n8n приглашает ввести email+password; это и есть регистрация |
| Кнопка Public не появляется | бесплатная версия неактивна | см. [10-common-errors.md §10.11](10-common-errors.md) |
| Установка зависла на unpacking | нормально, ждать | из урока: «Установка Data Bays подвисает, не страшно, наберитесь терпения» |
