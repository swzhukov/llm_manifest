## Профиль

| Поле | Значение |
|------|----------|
| **Название** | BIT.ASR (1bitai.ru) |
| **Тип** | ASR-сервис (распознавание речи) |
| **Домен** | `1bitai.ru` (поддомены: `bit-asr.1bitai.ru`, `bit-text-processor.1bitai.ru`) |
| **Связь** | **1С:Битрикс** (предположительно собственный продукт «Первого Бита» или партнёра) |

> **Гипотеза:** «1bitai» = «1С-Битрикс ИИ» или похожее. Это объясняет, почему Первый БИТ (франчайзи 1С:Битрикс) использует этот сервис.

## API Endpoints

| Endpoint | URL | Метод |
|----------|-----|-------|
| **Запуск транскрибации** | `https://bit-asr.1bitai.ru/start_transcribing` | POST |
| **Статус** | `https://bit-asr.1bitai.ru/get_status` | GET |
| **Получение файла** | `https://bit-asr.1bitai.ru/get_file` | GET |
| **Обработка текста** | `https://bit-text-processor.1bitai.ru/process-text` | POST |

## Параметры

| Параметр | Значение | Где используется |
|----------|----------|------------------|
| `diarize` | `true` | ASR (включить диаризацию) |
| `remove_timestamps` | `false` | ASR (сохранить метки времени) |

| `spelling` | — | Text Processor (орфография) |

| `accurate` | — | Text Processor (точность) |

- **ASR API:** заголовок `{"token": "<TOKEN>"}`
- **Text Processor API:** заголовок `{"Authorization": "Bearer <TOKEN>"}`

## Что работает, что нет (по наблюдениям Сергея)

| Что | Статус |
|-----|--------|
| Распознавание аудио | ✅ работает |

| Text Processor (пунктуация/орфография) | ⚠️ иногда возвращает пустые результаты |
| Fallback на оригинальный текст | ✅ реализован |

## Проблемы и решения (из чатов)

### Проблема 1: Диаризация не разделяет спикеров
- **Симптом:** API возвращает транскрипцию, но в ней один спикер.
- **Решение (в коде):** ручной парсинг + пост-обработка. Анализ качества: количество спикеров, сегментов, смен.

### Проблема 2: Text Processor возвращает пустоту
- **Решение:** гарантированное получение транскрипции даже при сбое Text Processor → повторные попытки → fallback на оригинальный текст → проверка качества (улучшенный текст должен быть не короче).

### Проблема 3: Обработка больших файлов
- **Решение:** оптимизация параметров аудио (моно, 16 kHz) перед ASR.

## Сравнение с Newton (по контексту использования)

| Параметр | Newton CLI | BIT.ASR |
|----------|-----------|---------|
| Движок | GigaAM v3 (Сбер) | Собственный (1bitai.ru) |
| Диаризация | Есть (но не идеально) | Есть, но ограниченно |
| Text Processor | Нет | Есть (пунктуация, орфография) |
| Аутентификация | Переменная окружения | Bearer token в заголовке |
| Стоимость | Токен | Токен (предположительно через 1С:Битрикс) |
| Где работает | Локально (CLI) + Python API | HTTP API (удалённо) |

**Что Сергей использует:**
- **Newton** — для n8n-телеграм-бота и YouTube-пайплайна.
- **BIT.ASR** — для более серьёзных задач (диаризация, пост-обработка).
