## Технические особенности, которые наверняка учтены (выводится из контекста)

| Особенность | Влияние на проект |
|-------------|-------------------|
| 2 типа продукции с разным циклом | Нужна разная модель учёта: серийная для УМК (короткий цикл) + штучная для надстроек (длинный цикл) |
| Отчётность по ГОЗ | Специфические требования к учёту, раздельный учёт по контрактам, контроль себестоимости |
| Пожарная техника и высотная техника | Скорее всего, модификации на одном шасси → нужна конфигурация продукции |
| Короткий цикл УМК (2 дня) | Высокая оборачиваемость → нужна быстрая приёмка/отгрузка, партионный учёт |

## Оценка Mavis

**Перспективы:** средние.

**За:**
- Клиент уже купил 1С:ERP (входной билет оплачен).
- Клиент понимает, что система не справляется → есть запрос на доработку.
- ГОЗ = государственные деньги, стабильный источник.
- Высотная/пожарная техника — нишевая, мало конкурентов.

**Против:**
- ГОЗ = жёсткая отчётность, штрафы за ошибки, бюрократия.
- Если у клиента уже есть «укрупнённая» настройка — её сначала надо разобрать, что политически сложно.
- Два разных цикла производства (УМК + надстройки) = сложная архитектура.
- Не раскрыты сроки и бюджет → непонятно, это «быстро закрыть дыры» или «большой проект на год».

**Рекомендация (если бы Сергей спросил):**
- **Сначала — гэп-анализ** (1-2 недели): что именно не работает в текущей ERP.
- **Пилот на УМК** (короткий цикл, проще): показать ценность быстро.
- **Затем — надстройки** (длинный цикл, ГОЗ): с привлечением эксперта по ГОЗ.
- **Отдельно — настройка отчётности ГОЗ**: это критично, штрафы могут убить репутацию.
- **В договоре** — фиксированная цена за пилот, потом time-and-material.
