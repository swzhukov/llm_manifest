## Цель системы

Создать персонального инвестиционного агента на базе n8n, который обеспечивает:

1. **Учёт всех инвестиционных сделок** через Telegram (текст/голос).
2. **Ежемесячную инвестиционную дисциплину** (план + напоминание).
3. **Регулярный отчёт по портфелю** с P&L и рекомендациями.
4. **Еженедельный дайджест инвестиционных сигналов** из Telegram-каналов и YouTube.

## Технический стек

| Компонент | Решение |
|-----------|---------|
| **Оркестратор** | n8n (self-hosted на Beget) |
| **Интерфейс** | Telegram Bot |
| **База данных** | Google Sheets |
| **LLM** | YandexGPT (основной агент + надсмотрщик) |
| **Голос** | Whisper API (транскрипция команд и видео) |
| **Котировки** | MOEX ISS API (бесплатно, без авторизации) |
| **YouTube мониторинг** | YouTube RSS Feed + YouTube Data API v3 |
| **RPA (опционально)** | Pix RPA — парсинг Альфа-Инвестиций |

## Google Sheets: 5 листов

### Лист 1: СДЕЛКИ
`дата | тип | блок | инструмент | тикер | количество | цена | сумма | брокер | счёт | комментарий`

Типы: покупка / продажа / пополнение_иис / купон / дивиденд / прочее

### Лист 2: ПОРТФЕЛЬ
`инструмент | тикер | блок | кол-во | цена_входа | сумма_входа | текущая_цена | текущая_стоимость | P&L_руб | P&L_% | доля_%`

Источник текущей цены:
- Биржевые (тикер есть на MOEX): автоматически через ISS API
- Прочие (ДУ, ВДО, нестандартные): ручной ввод раз в месяц

### Лист 3: ПАРАМЕТРЫ
```
целевая_доля_стабильный=40%
целевая_доля_рост=35%
целевая_доля_рискованный=25%
сумма_мес=15000
день_пополнения=1
иис_взнос_год_план=400000
иис_взнос_год_факт=[формула SUMIFS]
иис_вычет_потенциал=[=иис_взнос_год_факт*0.13]
```

### Лист 4: ДАЙДЖЕСТ_ЛОГ
`дата | источник | сигнал | тег | использован`

### Лист 5: YOUTUBE_КАНАЛЫ
`название | channel_id | rss_url | активен`

## Workflow 1: УЧЁТ СДЕЛКИ

**Триггер:** Telegram On Message (текст или голос)

**Шаги:**
1. Switch: тип сообщения (voice / text)
2. Если voice: HTTP POST → Whisper API → текст
3. **YandexGPT (агент)** — системный промпт:
   > «Ты парсер инвестиционных сделок. Из входящего текста извлеки JSON строго в формате: {"тип":"покупка|продажа|пополнение_иис|купон|дивиденд|прочее","блок":"стабильный|рост|рискованный","инструмент":"название","тикер":"тикер или null","количество":N,"цена":N,"сумма":N,"брокер":"название","счёт":"ИИС|брокерский|другой"}. Если сумма не указана — вычисли как кол-во×цену. Если данных не хватает — верни {"status":"need_more","question":"..."}»
4. IF: status == need_more → Telegram: задать вопрос, ELSE → Google Sheets: append в СДЕЛКИ
5. Telegram: подтверждение с форматированием

## Workflow 2: ЕЖЕМЕСЯЧНЫЙ ПЛАН

**Триггер:** Cron (1-е число каждого месяца, 09:00 MSK) + команда `/plan`

**Шаги:**
1. Google Sheets: читаем ПАРАМЕТРЫ и ПОРТФЕЛЬ
2. Рассчитать текущие доли по блокам
3. **YandexGPT** — системный промпт:
   > «Бюджет: {сумма_мес}₽. Текущие доли: стабильный {X}%, рост {Y}%, рискованный {Z}%. Целевые: 40/35/25. Рассчитай план на месяц: сколько рублей в каждый блок. Отдельно укажи статус ИИС: внесено {факт} из {план}, осталось {остаток}, потенциальный вычет {вычет}₽»
4. Telegram: отправить план

---

## Workflow 3: ОТЧЁТ ПОРТФЕЛЯ

**Триггер:** команда `/report` в Telegram

**Шаги:**
1. Google Sheets: читаем ПОРТФЕЛЬ
2. Loop по позициям с тикером:
   - HTTP GET: `https://iss.moex.com/iss/engines/stock/markets/shares/securities/{тикер}.json`
   - Парсим: LAST (последняя цена)
   - Обновляем текущую_цену в ПОРТФЕЛЬ
3. Рассчитать P&L, доли
4. **YandexGPT (надсмотрщик)** — системный промпт:
   > «Проверь данные. Если доля блока отклоняется от целевой более чем на 5% — дай краткую рекомендацию (1-2 предложения).»
5. Telegram: форматированный отчёт

## Workflow 4B: YOUTUBE-МОНИТОРИНГ

**Триггер:** Cron (каждый понедельник, 08:00) — запускается за час до 4A

**Каналы:** Николай Мрочковский, Евгений Коган (bitkogan), Smart-Lab, PRObonds (Иволга), ЦФА канал, InvestFuture

**Шаги:**
1. Google Sheets: читаем YOUTUBE_КАНАЛЫ
2. Loop по каналам:
   - RSS Feed Node: `https://www.youtube.com/feeds/videos.xml?channel_id={channel_id}`
   - Code Node (JS): фильтр НЕ Shorts (duration > 3 min), опубликовано ≤ 7 дней назад
   - YouTube Data API v3: статистика, теги
   - [Опционально] HTTP → Whisper: subtitle → транскрипт первых 500 слов
3. Merge всех результатов
4. YandexGPT: извлечь инвест-сигналы
5. Передать в WF4A

## Pix RPA (опционально, НЕ MVP)

**Назначение:** автоматический экспорт данных из приложения Альфа-Инвестиций (нет публичного API).

**Применение:** обновление текущей стоимости позиций в ДУ (ИИС «Точки Роста»).

**Триггер:** раз в месяц (перед формированием отчёта).

**Действие:** открыть Альфа-Инвестиции → скринировать экраны → передать данные через webhook в n8n → обновить лист ПОРТФЕЛЬ.

## Спорные / интересные моменты

### «Сначала» — дайджест, а не учёт
Сергей решил начать MVP с **Telegram-дайджеста** (WF4A), а не с учёта сделок (WF1). Логика: дайджест **сразу даёт ценность**, не требует исторических данных.

Это нестандартный приоритет — обычно начинают с «фундамента» (учёт). Сергей пошёл от **«быстрая ценность»**.

### YandexGPT, а не OpenAI
Использует YandexGPT, а не GPT-4/Claude. Причины (предположительно):
- Данные в РФ (нужна локализация, нет санкционных рисков).
- Стоимость (для MVP-этапа).
- Возможно, качество по русскому языку.

### Надсмотрщик (supervisor)
Классический паттерн «агент + надсмотрщик» (Zuevich). Один LLM-агент генерирует, второй проверяет и фильтрует. Это **снижает галлюцинации** и повышает качество.

## Оценка Mavis

**Перспективы:** высокие. ТЗ продуманное, стек реалистичный, есть примеры в сообществе.

**Риски:**
- **Pix RPA** для Альфа-Инвестиций — хрупкий, может сломаться при обновлении приложения. Нужен план Б (ручной ввод).
- **YandexGPT** может быть слабее GPT-4 на сложных финансовых формулировках. Стоит A/B-тестить.
- **Whisper** для русского голоса — иногда плохо распознаёт цифры («купил 100 акций Сургутнефтегаза» → может превратить в «купил 100 акций Сургут нефте газа»). Нужна пост-валидация.
- **MOEX ISS** иногда тормозит, нужны retry-механизмы.

**Что я бы рекомендовал (если бы Сергей спросил):**
- **Начать с WF4A** (как в плане) — это самая ценная быстрая победа.
- **YandexGPT промпт** — сделать 2-3 few-shot примера, чтобы повысить качество парсинга.
- **Google Sheets → Postgres** через 2-3 месяца, когда станет ясно, что система работает.
- **Pix RPA** — отложить, не делать в MVP. Пока — ручной ввод раз в месяц по ДУ.
