## Кейс 1: ERP-автоматизация группы компаний (2025-01-28)

### Задача продавца (Первого Бита)
Продать комплексный проект по автоматизации всей группы компаний агрохолдинга.

### Что обсуждалось
- ERP-система для нескольких юрлиц
- Этапы внедрения (водопад)
- Реестр разрывов (?)
- Лучшие практики, унификация процессов, прозрачность данных
- Бюджет

### Ошибки интегратора (по анализу LLM)
1. **Недостаточное понимание бизнес-потребностей** — фокус на техническом, не на бизнес-целях.
2. **Избыток технического жаргона** — «ERP», «Waterfall», «реестр разрывов» без пояснений.
3. **Отсутствие структуры в презентации** — перескакивали между темами.
4. **Игнорирование запроса на консалтинг** — клиент хотел экспертизу по процессам.
5. **Неготовность к возражениям** — общие фразы вместо аргументов.

### Что надо было делать
1. Discovery-анализ до встречи.
2. Связать IT с бизнес-целями («сокращение сроков согласования на 30%»).
3. Визуализация (схемы, BPMN).
4. Привлечь консультантов (KPMG/EY) для отраслевой экспертизы.
5. Кейсы и цифры («для холдинга X сократили с 5 дней до 4 часов»).

### Что делать сейчас
1. Workshop с фокусом на бизнес-задачи.
2. Адаптированное предложение с пилотом.
3. Включить отраслевого эксперта.
4. Storytelling (история успеха).
5. Demo на данных клиента.

## Кейс 2: Автоматизация финансового учёта (2025-02-17)

### Задача продавца
Продать проект по автоматизации финансового учёта для всей группы компаний.

### Что обсуждалось
- Бюджетирование (сейчас в Excel — главная боль)
- Моделирование
- Обмены данными
- Мэппинг, типовой функционал, гэпы
- Бюджет

### Ошибки интегратора
1. **Отсутствие структуры встречи** — начал с технического.
2. **Недостаток вопросов** — не выяснил текущие процессы.
3. **Технический жаргон** — «мэппинг», «типовой функционал», «гэпы».
4. **Нечёткий план проекта** — сроки, этапы, бюджет не согласованы.
5. **Игнорирование выгод** — не связал технику с бизнес-результатами.
6. **Неучтённые возражения** — клиент упомянул риски, интегратор не предложил решений.

### Что надо было делать
1. Начать с целей заказчика.
2. Методология «Проблема – Решение – Выгода».
3. Демонстрировать экспертизу через кейсы.
4. Поэтапный план.
5. Говорить на языке клиента.

### Что делать сейчас
1. Follow-up встреча с фиксацией приоритетов.
2. Структурированное КП.
3. Фокус на быстрых победах (пилот для одного направления).
4. Социальное доказательство (отзыв клиента).
5. Учесть возражения (Excel → автоматизация).

## Кейс 3: CRM-подбор (2025-02-19)

### Задача продавца
Подобрать оптимальное CRM-решение для заказчика (агрохолдинг, видимо).

### Что обсуждалось
- Интеграция телефонии (Манго, БиТ-телефония)
- Миграция данных из 1С
- Переход с УСН на ОСН (вопросы по налогообложению)
- Лицензирование 1С (УТ vs CRM)
- 15 рабочих мест
- Договор сопровождения

### Ошибки интегратора
1. **Недостаточная детализация по телефонии** — переадресовал коллегам, не дав чёткого ответа.
2. **Не прояснены детали миграции** — клиент упомянул проблему, продавец не предложил шагов.
3. **Слабая проработка налогообложения** — общие фразы про «настройку учёта».
4. **Нечёткое объяснение лицензирования** — клиент запутался в УТ vs CRM, продавец сослался на «правила 1С».

### Что сделано хорошо
1. Активно предложил демонстрацию.
2. Подчеркнул скорость поддержки (4 часа).
3. Успешно перевёл разговор на КП.
4. Профессионально собрал контакты.

### Что делать сейчас
1. Срочно отправить письмо с детализацией.
2. Организовать демо с техническим специалистом.
3. Пилотный проект (2 недели тестового доступа).
4. Аргумент «скорой поддержки» в письме.

## Кейс 4: Репрайсер для маркетплейсов (2025-02-14)

> Это был **продуктовый тренинг** по продукту «Репрайсер» — инструмент для селлеров на маркетплейсах (автоматическое ценообразование).

### Что за продукт
**Репрайсер** — инструмент для автоматического ценообразования на маркетплейсах (Wildberries, Ozon и др.). Анализирует конкурентов, остатки, спрос. Устанавливает оптимальную цену.

### Стратегии ценообразования
1. **Эмель-оптимизатор** — «золотая середина» между ценой и спросом.
2. **Следование за конкурентами** — на уровне минимальной цены или медианы.
3. **Контроль остатков** — повышение при дефиците, снижение при избытке.
4. **Распродажа** — снижение для оборота.

### Контексты селлеров
- Большой ассортимент (5000+ товаров).
- Конкурентные ниши (демпинг).
- Сезонные товары / акции.
- Дефицит/избыток товара.
- Новички.
- Уникальные товары.

### Скрипты продаж
- «Защитите себя от убытков» (новички).
- «Увеличьте прибыль без увеличения рабочего дня» (середнячки).
- «Управляйте 10 000 товаров без стресса» (крупные).

### Возражения и ответы
- «Не верю в автоматизацию» → пробный период.
- «Дорого» → окупаемость через рост маржи.
- «Уникальные товары» → аналитика спроса.
- «Мало товаров» → минимальная цена как базовая защита.
- «Сложно настроить» → бесплатная настройка.
- «Акции портят» → Репрайсер учитывает.
