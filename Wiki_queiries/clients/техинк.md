## Что внедряем

**1С:ERP** — для полноценного учёта, контроля, управления и **отчётности по ГОЗ** (Государственный оборонный заказ).

---

## Текущее состояние у клиента

- 1С:ERP **уже есть** (входной билет оплачен).
- Используется **укрупнённо** для пожарной и высотной техники.
- **Не удовлетворяет** информационные потребности:
  - Полноценный учёт
  - Контроль
  - Управление
  - **Сдача отчётности ГОЗ** ← это критично

---

## Ключевые вызовы

| Вызов | Описание |
|-------|----------|
| **2 типа продукции с разным циклом** | Нужна разная модель учёта: серийная для УМК (2 дня) + штучная для надстроек (длинный цикл) |
| **ГОЗ отчётность** | Жёсткие требования, штрафы за ошибки, бюрократия |
| **Модификации на одном шасси** | Высотная/пожарная техника, вероятно, имеют общее шасси → нужна конфигурация продукции |
| **Короткий цикл УМК** | Высокая оборачиваемость → нужна быстрая приёмка/отгрузка |

## Перспективы проекта (оценка Mavis)

| Фактор | Оценка |
|--------|--------|
| **Клиент уже на 1С:ERP** | ✅ вход оплачен, миграция минимальна |
| **Клиент понимает проблему** | ✅ сам говорит «не удовлетворяет» |
| **ГОЗ = стабильные деньги** | ✅ государственный контракт, не зависит от рынка |
| **Нишевая отрасль** (пожарная/высотная техника) | ✅ мало конкурентов, барьеры входа |
| **Штрафы за ошибки в ГОЗ** | ⚠️ высокий риск, нужен эксперт |
| **Два цикла производства** | ⚠️ сложная архитектура |
| **Бюджет/сроки неясны** | ⚠️ невозможно планировать |
| **Команда Первого Бита не названа** | ⚠️ неясно, хватит ли экспертизы |

**Общая оценка:** средние перспективы. Проект реализуем, но **нужны вводные** (бюджет, сроки, текущие гэпы), чтобы дать детальный план.

## Что я бы рекомендовал Сергею

1. **Запросить транскрипт встречи** для детального разбора.

4. **Затем надстройки + ГОЗ-отчётность** (с экспертом).
5. **Фикс-прайс** на пилот, time-and-material на остальное.
