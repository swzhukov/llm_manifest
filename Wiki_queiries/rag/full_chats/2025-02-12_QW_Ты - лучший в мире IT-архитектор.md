## AI

### Функционально-технические требования  
---

#### **1. Хранение ограничений**  
**1.1. Постоянные ограничения (весогабаритные):**  
- Справочник "Ограничения складов Ozon" с полями:  
  - Склад (привязка к справочнику "Склады Ozon").  
  - Максимальный вес на поставку (кг).  
  - Максимальный объем (м³).  
  - Категории товаров (если ограничения категорийные).  
- Данные заполняются вручную и обновляются при изменении условий Ozon.  

**1.2. Временные ограничения:**  
- Регистр сведений "Динамические ограничения Ozon" с полями:  
  - Склад.  
  - Дата начала/окончания ограничения.  
  - Категория товара.  
  - Лимит (количество, вес, объем или запрет).  
- Автоматическая синхронизация с API Ozon (REST/JSON) 1 раз в час.  

**1.3. Тайм-слоты доступности складов:**  
- Интеграция с API Ozon (метод `getTimeSlots`).  
- Хранение данных в регистре сведений "Доступность складов" с полями:  
  - Склад.  
  - Дата и время слота.  
  - Статус (доступен/недоступен).  
- Проверка доступности перед созданием поставки.  

---

#### **2. Алгоритм распределения поставок**  
**2.1. Создание документа "План поставки":**  
- Документ содержит:  
  - Номенклатура.  
  - Количество.  
  - Вес и объем (авторасчет на основе справочника номенклатуры).  
  - Целевой склад (предварительно выбранный пользователем).  

**2.2. Проверка ограничений:**  
Для каждой строки поставки:  
- Проверка постоянных ограничений склада (вес, объем).  
- Проверка временных ограничений (категории, даты).  
- Проверка доступности тайм-слота (через API).  

**2.3. Автоматическое перераспределение:**  
- Если склад не подходит:  
  - Поиск альтернативных складов по приоритету:  
    - Совпадение категории товара.  
    - Наличие доступных тайм-слотов.  
    - Минимальная загруженность (на основе исторических данных).  
  - Создание новой поставки на подходящий склад.  

**2.4. Ручное вмешательство:**  
- Если алгоритм предлагает несколько вариантов, пользователь выбирает склад вручную.  
- Возможность переопределения автоматических решений.

### Архитектурное решение на базе 1С:Управление торговлей  
#### **Компоненты:**  
1. **Справочники:**  
   - Склады Ozon.  
   - Ограничения складов (постоянные и временные).  

2. **Документы:**  
   - План поставки.  
   - Результат распределения (связь с исходным планом).  

3. **Регистры сведений:**  
   - Динамические ограничения.  
   - Доступность тайм-слотов.  

4. **Внешние подключения:**  
   - REST-клиент для интеграции с API Ozon.  
   - Обработки для синхронизации данных.  

5. **Отчеты:**  
   - Лог проверки ограничений.  
   - Анализ загруженности складов.
