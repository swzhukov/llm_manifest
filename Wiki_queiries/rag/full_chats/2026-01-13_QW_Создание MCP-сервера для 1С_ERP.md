## AI

Создание MCP-сервера (Model Context Protocol) для проектов на 1С:ERP позволяет подключать AI-ассистенты (как Claude, Cursor, GPT) к вашей базе данных, чтобы они могли работать с её данными и метаданными. Есть несколько путей: готовые решения, разработка своего сервера или использование open-source проектов.

Вот основные подходы и шаги для реализации:

| Подход | Описание | Лучше всего подходит для | Пример / Ссылка |
| :--- | :--- | :--- | :--- |
| **Готовое решение** | Покупка и установка коммерческого сервера. Быстрый старт, техническая поддержка, безопасность. | Компаний, которым нужно быстро внедрить функционал без глубокой разработки. | MCP-сервер от ARQA (поддерживает 1С:ERP 2.4/2.5) [citation:7]. |
| **Разработка своего MCP-сервера** | Создание сервера с нуля на Python, Go и др. Полный контроль над логикой и данными. | Разработчиков, которым нужна кастомная логика или глубокое понимание процесса. | Подробная статья на Infostart с пошаговым руководством [citation:1]. |
| **Open-source проект** | Использование готового кода, адаптация под свои задачи. Баланс между скоростью и контролем. | Разработчиков, готовых изучать и дорабатывать чужой код. | Сервер на Go для 1С ERP [citation:9] или проект с Docker и Qdrant [citation:4]. |

### 🔧 Практические шаги по созданию и подключению MCP-сервера
Независимо от выбранного пути, общая архитектура и шаги будут схожими:

1.  **Определите функции (Tools) сервера**
    Решите, что должен уметь ваш AI-помощник: запрашивать данные (остатки, продажи), анализировать метаданные конфигурации или создавать документы. Это определит логику сервера.

2.  **Выберите технологию и создайте MCP-сервер**
    Сервер выступает **посредником** между AI и 1С[citation:1]. Его можно написать на Python, Go, Node.js. Основная задача — принимать запросы от модели, обращаться к 1С (через HTTP-сервисы, COM или OLE) и возвращать данные в удобном формате.
    *   **Пример из open-source**: Проект [ruslan-hut/onec-mcp](https://github.com/ruslan-hut/onec-mcp) на Go реализует поиск контрагентов, складов и формирование отчетов о продажах [citation:9].
    *   **Пример для метаданных**: В статье [citation:1] и инструменте [citation:4] описан сервер, который подключается к базе 1С как расширение и предоставляет AI структуру метаданных (справочники, документы), что критически важно для написания корректного кода.

3.  **Обеспечьте безопасный доступ к 1С:ERP**
    Сервер должен авторизовываться в 1С от имени пользователя с ограниченными правами. Используйте безопасное соединение (TLS) и контролируйте, к каким данным и операциям есть доступ [citation:7].

4.  **Подключите сервер к AI-инструменту**
    Современные IDE и AI-клиенты поддерживают MCP. Нужно указать в настройках проекта путь к вашему серверу:
    *   **Для Cursor**: Создайте файл `.cursor/mcp.json` в корне проекта.
    *   **Для VS Code + Copilot**: Создайте файл `.vscode/mcp.json`.
    Пример конфигурации для подключения к локальному серверу, работающему по HTTP:
    ```json
    {
        "mcpServers": {
            "my-1c-erp-server": {
                "url": "http://localhost:8000/mcp"
            }
        }
    }
    ```
    Более подробные примеры для разных сред можно найти в инструкциях к проекту [citation:4].

### 💡 Рекомендации и важные нюансы
*   **Начните с метаданных**: Для задач разработки (например, в Cursor) самый полезный первый шаг — сделать MCP-сервер, который предоставляет AI структуру вашей конфигурации 1С:ERP. Это резко повысит качество генерируемого кода [citation:1][citation:4].
*   **Используйте RAG для сложных конфигураций**: Для больших и нестандартных конфигураций эффективно использовать векторную базу данных (RAG). Как в проекте [citation:4], где описания объектов хранятся в Qdrant, что позволяет AI точно находить нужные сущности по запросу.
*   **Безопасность данных**: При использовании облачных AI-моделей важно не передавать им сырые чувствительные данные. MCP-сервер должен отправлять только необходимые агрегированные данные или результаты запросов [citation:2][citation:7].

Если вы выберете путь самостоятельной разработки, на каких аспектах — безопасность, работа с метаданными или интеграция с конкретной AI-средой — вам хотелось бы узнать больше?
