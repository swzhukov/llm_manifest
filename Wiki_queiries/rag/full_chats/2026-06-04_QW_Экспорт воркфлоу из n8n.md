# Экспорт воркфлоу из n8n

## USER

А можно как-то экспортировать свой воркфлоу из n8n?

## AI

Да, экспортировать воркфлоу из n8n можно, и сделать это можно несколькими способами: от самого быстрого ручного до полностью автоматизированного.

### 🚀 1. Самый простой способ: Копирование и вставка

Этот метод идеален для быстрого переноса небольших воркфлоу между инстансами или для обмена с коллегами.

*   **Экспорт:** Откройте ваш воркфлоу в редакторе n8n. Выделите все узлы на канве (можно нажать `Ctrl+A` / `Cmd+A`), скопируйте их (нажав `Ctrl+C` / `Cmd+C`), а затем вставьте в текстовый файл.
*   **Импорт:** Просто вставьте скопированные данные на канву нового воркфлоу (нажав `Ctrl+V` / `Cmd+V`)[reference:0][reference:1].

Этот способ создаст "сырой" JSON, из которого безопасность данных ложится на вас. Он не подходит для переноса больших объемов, но очень удобен для небольших проектов.

### 💻 2. Экспорт через меню редактора

Если вам нужно сохранить точную копию текущего воркфлоу, проще всего воспользоваться встроенным меню.

*   **Инструкция:** В верхней панели редактора нажмите на три точки (`...`) и выберите пункт **"Download"** — ваш воркфлоу будет сразу сохранен на компьютер в виде JSON-файла[reference:2][reference:3].
*   **Обратный импорт:** На той же панели выберите **"Import from File"** и укажите нужный JSON-файл[reference:4].

### 🐳 3. Экспорт для профессионалов: Командная строка (CLI)

Этот метод незаменим для миграции, резервного копирования и автоматизации. Он подходит для **самостоятельно установленного (self-hosted) экземпляра** n8n.

*   **Экспорт всех воркфлоу:** Чтобы сразу выгрузить всё:
    ```bash
    n8n export:workflow --all --output=all_workflows.json
    ```
*   **Экспорт конкретного воркфлоу:** Если нужен только один, укажите его ID:
    ```bash
    n8n export:workflow --id=ID_ВАШЕГО_ВОРКФЛОУ --output=my_workflow.json
    ```
*   **Импорт:** Чтобы восстановить из файла:
    ```bash
    n8n import:workflow --input=my_workflow.json
    ```

Если вы используете Docker, команды нужно выполнить внутри контейнера. Это выглядит немного сложнее, но дает полный контроль[reference:5]:

```bash
# Экспорт
docker exec -it n8n n8n export:workflow --all --output=/home/node/workflows.json
# Копирование из контейнера
docker cp n8n:/home/node/workflows.json ./workflows.json

# Импорт
docker cp ./workflows.json n8n:/home/node/workflows.json
docker exec -it n8n n8n import:workflow --input=/home/node/workflows.json
```

### ☁️ 4. Экспорт для n8n Cloud

Владельцы аккаунтов в облачном сервисе могут скачать резервную копию всех своих воркфлоу через **Панель администратора (Admin Dashboard)**. В разделе **"Manage"** выберите вкладку **"Export"** и нажмите **"Download Workflows"**[reference:6].

### 🔧 5. Экспорт через API

Технически опытные пользователи могут автоматизировать экспорт с помощью **REST API** n8n. Это позволит легко встроить задачу создания бэкапов в ваш собственный воркфлоу.

### ⚠️ Важное предупреждение о безопасности

При любом экспорте помните, что в JSON-файле могут содержаться конфиденциальные данные:

*   **Имена и ID учетных данных (Credentials).** Сами пароли и ключи не экспортируются, но их имена могут раскрыть, с какими сервисами вы работаете[reference:7].
*   **HTTP-заголовки.** Если вы импортировали узел из cURL, в файле могут сохраниться заголовки авторизации в открытом виде[reference:8].

При использовании CLI для экспорта учетных данных предусмотрен специальный флаг `--decrypted`, который экспортирует их в незашифрованном виде. Это очень удобно для переноса между инстансами, но с таким файлом нужно обращаться максимально осторожно — это **чувствительные данные**[reference:9].
