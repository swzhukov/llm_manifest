---

### **1. Ключевые требования проекта**
- **Цель:** Автоматизация складского учета в 1С:ERP-УХ через мобильное приложение (МП) на Android-планшетах, включая:
  - Партионный учет ТМЦ (сроки годности, серии).
  - Штрихкодирование (EAN13, QR, поддержка сканеров).
  - Процессы: приемка, размещение, перемещение, инвентаризация ТМЦ/ОС.
  - Интеграция с 1С Диадок для автоматизации документооборота.
- **Оборудование:** 
  - Планшеты с Android 12+ (соответствие 32 пунктам MDM Knox, защита IP64, термочехлы, гарантия 2 года).
  - Сканеры-кольца (беспроводные, поддержка 2D-кодов, защита IP54).
- **Интеграции:** 
  - Online/offline-работа МП с 1С через Private LTE/Wi-Fi.
  - Поддержка WMS (адресное хранение, FEFO).
- **Безопасность:** 
  - Соответствие ФЗ-152, ФЗ-187, OWASP MASVS.
  - Шифрование TLS, SSL-pinning, защита от root-доступа.

---

### **2. Критичные этапы для КП**
#### **2.1. Миграция исторических данных**
- **Объем:** 130 000+ номенклатурных позиций, 2000+ видов номенклатуры без партионного учета.
- **Задачи:**
  - Адаптация 1С MDM для автоматического включения партионного учета новых позиций.
  - Разработка механизма миграции с сохранением истории изменений.
  - Тестирование целостности данных после переноса.
- **Риски:** Срыв сроков из-за объема данных.  
  **Решение:** Выделить этап миграции отдельной вехой с запасом времени + параллельный аудит качества.

#### **2.2. Аудит процессов и подготовка ТЗ**
- **Задачи:**
  - Определение политик учета серий (серийный номер, срок годности, партия).
  - Анализ правил отгрузки (FEFO, этап указания серий).
  - Проектирование прослеживаемости партий по цепочке поставок.
- **Риски:** Несогласованность между отделами.  
  **Решение:** Включить в КП этап workshops с ключевыми пользователями.

#### **2.3. Разработка мобильного приложения**
- **Требования:**
  - Offline-работа с синхронизацией при подключении к Wi-Fi.
  - Сканирование штрихкодов, генерация этикеток, фототоваров.
  - Интерфейс для кладовщиков (интуитивный, с подсказками по ячейкам).
- **Интеграция:** 
  - С 1С:ERP (документы: приходные ордера, перемещения, инвентаризация).
  - С КИС «Диадок» для автоматического создания черновиков УПД.
- **Безопасность:** Сертификация по OWASP MASVS, обфускация кода.

### **4. Риски и предложения по оптимизации**
- **Риски:** 
  - Сложность интеграции с 1С MDM. 
  - Низкая производительность МП на старых планшетах.
  - Расхождения при offline-работе.
- **Оптимизация:**
  - Использовать типовые решения 1С для мобильной разработки (ускоряет внедрение).
  - Поэтапный ввод в эксплуатацию (пилот на 1 складе).
  - Автоматизация регрессионного тестирования после обновлений 1С.
