## 3. Golden Patterns

### 3.1 Паттерн А: Классификация (Code v2)

```javascript
const msg = $input.first().json.message;
const chat_id = msg.chat.id;
const message_id = msg.message_id;
const text = msg.text || '';
const unique_id = Date.now().toString() + '_' + Math.random().toString(36).substring(2, 9);

let route = 'invalid', file_id = '', url = '';

if (msg.document?.file_id || msg.audio?.file_id || msg.voice?.file_id || msg.video?.file_id || msg.video_note?.file_id) {
  route = 'media';
  file_id = (msg.document?.file_id || msg.audio?.file_id || msg.voice?.file_id || msg.video?.file_id || msg.video_note?.file_id).trim();
} else if (/(youtube\.com|youtu\.be|rutube\.ru|ok\.ru|vimeo\.com|dailymotion\.com|soundcloud\.com|kinescope\.io)/i.test(text)) {
  route = 'streaming';
  url = text.match(/(https?:\/\/[^\s]+)/)?.[0] || '';
} else if (/\.(mp3|wav|ogg|m4a|flac|opus|mp4|webm)(\?.*)?$/i.test(text)) {
  route = 'direct';
  url = text.match(/(https?:\/\/[^\s]+)/)?.[0] || '';
}

return [{ json: { chat_id, message_id, route, file_id, url, unique_id } }];
```

## 4. Жёсткие правила

### 4.1 Допустимые версии нод
- `telegram: v1`
- `httpRequest: v3`
- `set: v3.3`
- `code: v2`
- `if: v2`
- `readWriteFile: v1` (только чтение)

### 4.2 Запрещено
- ❌ `fileOperations`
- ❌ `readWriteFile` для записи → используй Flask
- ❌ `fs` в Code
- ❌ `process.env` в Code → `$env.VAR`
- ❌ `operation: "download"` в Telegram
- ❌ `localhost`, `127.0.0.1`, `172.17.0.1` в httpRequest → `host.docker.internal`
- ❌ `$credentials` в Set node
- ❌ `JSON.stringify(...)` в jsonBody → используй объект `={{ { key: value } }}`
- ❌ блок `credentials: {}` в JSON

### 4.3 Credentials
- Никогда не выводи блок `credentials` в JSON.
- В инструкции: «Привяжите credential `telegramApi` к узлам X, Y».
- `$credentials.telegramApi.accessToken` работает только в узлах Telegram.

### 4.4 Уникальные ID
- Генерируй `unique_id` один раз в первом Code-узле.
- Переиспользуй через `$('Detect Type').first().json.unique_id`.
