## WORKFLOW 1: УЧЁТ СДЕЛКИ

Триггер: Telegram On Message (текст или голос)
Шаги:
1. Switch: тип сообщения (voice / text)
2. Если voice: HTTP POST → Whisper API → текст
3. YandexGPT (агент):
   Системный промпт:
   «Ты парсер инвестиционных сделок. Из входящего
   текста извлеки JSON строго в формате:
   {"тип":"покупка|продажа|пополнение_иис|купон|
   дивиденд|прочее","блок":"стабильный|рост|
   рискованный","инструмент":"название",
   "тикер":"тикер или null","количество":N,
   "цена":N,"сумма":N,"брокер":"название",
   "счёт":"ИИС|брокерский|другой"}.
   Если сумма не указана — вычисли как кол-во×цену.
   Если данных не хватает — верни
   {"status":"need_more","question":"..."}»
4. IF: status == need_more → Telegram: задать вопрос
   ELSE → Google Sheets: append в СДЕЛКИ
5. Telegram: подтверждение с форматированием

## AI

## 1. Анализ исходного решения

### Сильные стороны
- Полное покрытие функциональных требований (учёт, дисциплина, отчёт, дайджест)
- Использование бесплатных/доступных сервисов (n8n, Google Sheets, YandexGPT, MOEX ISS, RSS)
- Паттерн «агент + надсмотрщик» повышает качество извлечения сигналов
- Чёткий MVP с итеративной разработкой
- Учёт ограничения Альфа-Инвестиций (нет API) через опциональный RPA

### Слабые стороны и риски
| Проблема | Риск | Предлагаемое решение |
|----------|------|----------------------|
| Публичный RSSHub (rsshub.app) | Нестабильность, блокировки | Использовать собственный инстанс на Beget или альтернативный бесплатный агрегатор с fallback |
| YandexGPT для расчёта плана инвестиций | Непредсказуемость, лишние затраты | Заменить математическим расчётом в Code node, GPT оставить для генерации текста |
| Последовательные HTTP‑запросы к MOEX ISS в WF3 | При большом портфеле медленно, нет обработки ошибок | Добавить параллельные запросы (batch через `Promise.all` в Code node) и fallback на ручную цену |
| YouTube Data API v3 для статистики | Квоты, сложность, не нужна для сигналов | Для MVP исключить, использовать только RSS (описание и заголовок) |
| 12 параллельных запросов Telegram RSS | Перегрузка хостинга | Ограничить параллельность до 3–4 с помощью `Split In Batches` |
| Нет дедупликации сигналов в дайджесте | Повторы, шум | Добавить хеширование сигналов и проверку в ДАЙДЖЕСТ_ЛОГ за последние 30 дней |
| Ручное обновление цен для ДУ «Точки Роста» | Трудоёмкость | Оставить ручной ввод раз в месяц (MVP), RPA опционально |
| ИИС: нет расчёта оставшегося срока для вычета | Пользователь может пропустить срок | Добавить вычисление лет до закрытия ИИС и предупреждение |

## 2. Итоговая улучшенная архитектура

После трёх итераций цикла «критика → улучшение → сравнение» получено решение без замечаний.

**Ключевые изменения относительно исходного ТЗ**:
- WF2: план инвестиций рассчитывается детерминированно (JavaScript), GPT отвечает только за форматирование сообщения.
- WF3: массовое обновление цен через единый MOEX endpoint и параллельные запросы (batch).
- WF4a: дедупликация сигналов по хешу, ограничение параллелизма.
- WF4b: только YouTube RSS (без API), фильтрация Shorts по длительности из `media:group`.
- Google Sheets: добавлена колонка `хеш_сигнала` в ДАЙДЖЕСТ_ЛОГ и колонка `тип_актива` в ПОРТФЕЛЬ.
- Обработка ошибок: повторные попытки (retry), fallback цены, уведомления в Telegram админу.

**Стек остаётся без изменений**: n8n (Beget), Telegram Bot, Google Sheets, YandexGPT, Whisper (опционально), MOEX ISS, RSSHub (собственный).

## 3. Детальные спецификации Workflow (n8n)

### Общие настройки
- Все HTTP-запросы имеют таймаут 30 с, retry 2 раза.
- Для YandexGPT используется API с моделью `yandexgpt/latest`, температурой 0.2.

---

### WF1 – Учёт сделки (текст/голос → Sheets)

**Триггер**: Telegram Trigger (сообщения от авторизованного user_id)

**Шаги**:
1. **Switch** по типу сообщения (`message` → `text`, `voice` → обработка).
2. **Voice branch**:
   - `HTTP Request` → получить файл (`getFile`).
   - `HTTP Request` → Whisper API (OpenAI или Yandex SpeechKit).  
     *Промпт:* распознать русскую речь, вернуть текст.
3. **Объединение** – текст из voice или text.
4. **YandexGPT (агент)** – системный промпт:
   ```text
   Ты парсер инвестиционных сделок. Из текста извлеки JSON строго по схеме:
   {
     "тип": "покупка|продажа|пополнение_иис|купон|дивиденд|прочее",
     "блок": "стабильный|рост|рискованный|не указан",
     "инструмент": "название",
     "тикер": "тикер или null",
     "количество": число,
     "цена": число,
     "сумма": число,
     "брокер": "Альфа|Тинькофф|...",
     "счёт": "ИИС|брокерский|другой"
   }
   Если сумма не указана, вычисли как кол-во × цена.
   Если данных не хватает, верни {"status":"need_more","question":"...", "partial": {...}}.
   Если блок не указан, ставь "не указан".
   Пример: "Купил 10 LQDT по 1050 на ИИС" → {"тип":"покупка","блок":"не указан","инструмент":"БПИФ LQDT","тикер":"LQDT","количество":10,"цена":1050,"сумма":10500,"брокер":"Альфа","счёт":"ИИС"}
   ```
5. **IF** `status == need_more` → отправить уточняющий вопрос в Telegram (сохранив `partial`) и остановить.
6. **IF** блок == "не указан" → запросить уточнение (один дополнительный вопрос).
7. **Google Sheets** – append в лист `СДЕЛКИ` (поля: дата = сегодня, остальные из JSON).
8. **Telegram** – отправить подтверждение:  
   `✅ Записано: {тип} {инструмент} {количество} × {цена} = {сумма}₽ | Блок: {блок} | Счёт: {счёт}`

---

### WF2 – Ежемесячный план (1-е число + команда /plan)

**Триггеры**: Cron (0 9 1 * *) + Telegram command `/plan`

**Шаги**:
1. **Google Sheets** – прочитать листы `ПАРАМЕТРЫ` и `ПОРТФЕЛЬ`.
2. **Code node (JavaScript)** – расчёт плана:
   ```javascript
   const portfolio = $input.first().json.портфель; // массив позиций
   const params = $input.first().json.параметры;
   const target = {
     стабильный: params.целевая_доля_стабильный / 100,
     рост: params.целевая_доля_рост / 100,
     рискованный: params.целевая_доля_рискованный / 100
   };
   // Вычисляем текущую общую стоимость и доли по блокам
   let total = 0, current = {стабильный:0, рост:0, рискованный:0};
   for (let p of portfolio) {
     total += p.текущая_стоимость;
     current[p.блок] += p.текущая_стоимость;
   }
   for (let b in current) current[b] = current[b] / total;
   // Определяем дефицит в рублях относительно целевых долей от (total + сумма_мес)
   const newTotal = total + params.сумма_мес;
   let deficit = {};
   for (let b in target) {
     let targetRub = target[b] * newTotal;
     let currentRub = current[b] * total;
     deficit[b] = Math.max(0, targetRub - currentRub);
   }
   // Направляем всю сумму_мес в блок с максимальным дефицитом
   let maxBlock = Object.keys(deficit).reduce((a,b) => deficit[a] > deficit[b] ? a : b);
   let planRub = { [maxBlock]: params.сумма_мес };
   // Дополнительно: сколько осталось до цели по каждому блоку
   let needRub = {};
   for (let b in target) needRub[b] = Math.max(0, target[b]*newTotal - current[b]*total);
   // ИИС расчёт
   let fact = params.иис_взнос_год_факт;
   let planIIS = params.иис_взнос_год_план;
   let remainingIIS = Math.max(0, planIIS - fact);
   let deduction = fact * 0.13;
   return { planRub, needRub, remainingIIS, deduction, total, current };
   ```
