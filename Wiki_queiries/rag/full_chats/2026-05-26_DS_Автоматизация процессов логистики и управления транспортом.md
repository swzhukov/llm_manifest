Схема процессов:


## 1. Общая архитектура процессов

Все ваши процессы вращаются вокруг **организации и проведения мероприятий / образовательных заездов** (ДКПП, ДСТИ, образовательные заезды). Выделяются три уровня:

| Уровень | Содержание | Примеры процессов |
|--------|------------|-------------------|
| **Стратегический / Программный** | Жизненный цикл мероприятия от идеи до отчёта | Карты процессов ДКПП, ДСТИ, образовательные заезды; Схема реализации программы |
| **Операционный (обеспечение мероприятия)** | Договорная работа, логистика участников, кадровое обеспечение | Заключение договора ГПХ, Договор с физ. лицом, Релокация, Подбор сотрудника |
| **Сервисный / Контролирующий** | Взаимодействие с участниками, протокольное сопровождение, проверки безопасности | Сл. заботы, Сл. протокольного сопр., СБ, Финансы, Юристы |

**Базовые системы (IT-ландшафт):** 1С (учёт и согласования), КЭДО (кадровый документооборот), Контур-Сайн (ЭДО для договоров), Portal.orki.art (внешний портал для участников/партнёров).

## 2. Детальные блок-схемы (по файлам)

### 2.1. Служба заботы об участниках — взаимодействие
**Ответственные:** Сл. заботы, Сл. безопасности, Юристы, Сл. протокольного сопровождения.

**Логика:**
- **Начало** (зелёный круг) → Заявка в АИС → Проверка безопасности:
  - если НЕТ → конец (красный круг).
  - если ДА → Проверка заявки на соцсети/телеграм (Служба заботы).
- **Ветвление:** 
  - При отсутствии конфликтов → Передача на подписание в Юридическую службу.
  - При наличии вопросов → Письмо кандидату с просьбой предоставить подтверждающие документы → Проверка документов.
- **Контрольные точки:**
  - Подтверждение готовности (Служба протокольного сопровождения).
  - Финал: Получение билета/проживания → Конец.

### 2.2. Договор с физическим лицом (в рамках ДСТИ)
**Участники:** Контрагент (физическое лицо), СБ, Кадровая служба, Руководитель департамента, ЭТА (вероятно, Экспертно-техническая аттестация?), Финансовая служба, Бухгалтерия, Юридическая служба.

**Этапы:**
1. Контрагент создаёт заявку в 1С (или через кадровую службу).
2. **Цепочка согласований (каждый орган проверяет заявку):**
   - СБ → проверка благонадёжности (если НЕТ → возврат на доработку).
   - Кадровая служба → проверка соответствия должности/проф. стандартам (если НЕТ → возврат).
   - Руководитель департамента → согласование необходимости (если НЕТ → возврат).
   - ЭТА → согласование (если НЕТ → возврат).
   - Финансовая служба → проверка бюджета (если НЕТ → возврат).
   - Бухгалтерия → проверка реквизитов и соответствия законодательству.
3. После всех согласований: **Подготовка договора** (Юридическая служба). Размещение договора в ЭДО.
4. **Подписание договора** (сначала контрагентом, затем подписывает Юридическая служба).
5. Размещение подписанного договора в 1С → финал.

### 2.3. ДСТИ взаимодействие (подготовка образовательных заездов)
**Участники:** Служба заботы, Служба по работе с партнёрами, Служба протокольного сопровождения, Служба безопасности, Служба по работе с участниками, Служба методического сопровождения, ИТ, Служба организации питания, Служба протокольного сопр. (отдельные ветки).

**Основные ветки:**
- **Ветка «Заезд участников»:**
  - Служба заботы: Проверка документов участников (паспорт, билеты, медсправки).
  - Если ошибки → Письмо о необходимости исправления.
  - Если ДА → Формирование списка заезда → Служба протокольного сопровождения подтверждает.
- **Ветка «Организация мероприятия»:** 
  - Служба по работе с партнёрами: Подготовка инфраструктуры (площадки, техническое оснащение).
  - Служба безопасности: Проверка списков на соответствие требованиям.
- **Ветка «Образовательная программа»:**
  - Служба методического сопровождения: Подготовка учебных материалов, расписания.
  - Служба организации питания: Обеспечение обедов, кофе-брейков.
- **Завершающая стадия:** Закрытие мероприятия, подведение итогов, передача данных в 1С.

### 2.4. Заключение договора ГПХ (гражданско-правового характера)
**Системы:** Portal.orki.art, 1С, Контур-Сайн.

**Роли:** Сотрудник (физическое лицо), Тендерная служба (возможно, отдел закупок), Секретарь тендерной комиссии, Юридическая служба, Руководитель департамента, Бухгалтерия, Финансовый контролёр, Подразделение по управлению проектами.

**Процесс:**
1. **Сотрудник / контрагент:** Вход на портал, заполнение формы (реквизиты, паспорт, ИНН).
2. **1С:** Автоматическая проверка контрагента (сервис ФНС, судебные задолженности):
   - Если проверка не пройдена → красный круг (отказ).
   - Если пройдена → Создание заявки в 1С.
3. **Согласование заявки:**
   - Тендерная служба → проверка обоснованности выбора контрагента.
   - Юридическая служба → проверка формы договора ГПХ, отсутствие признаков трудового договора.
   - Руководитель департамента → подтверждение необходимости и соответствия бюджета.
   - Бухгалтерия → проверка ставок и налогов.
   - Финансовый контролёр → контроль лимитов финансирования.
4. **Подготовка договора:** Юридическая служба формирует проект.
5. **Электронное подписание (Контур-Сайн):**
   - Контрагент получает ссылку, подписывает.
   - Юридическая служба подписывает с другой стороны.
6. **Заключённый договор** → Выгрузка в 1С → Завершение.

### 2.5. Карта процессов ДКПП (вероятно, «Деловая коммуникация и проектная работа»)
Содержит **6 этапов** (по вертикали):
1. **Планирование проекта** (инициация, формирование команды).
2. **Реализация проекта** (подэтапы: 2.1. Запуск, 2.2. Сопровождение жизни смены, 2.3. Промежуточная аттестация, 2.4. Медиа, 2.5. Работа с партнёрами, 2.6. Управление качеством, 2.7. Пост-событие, 2.8. Подведение итогов).
3. **Подготовка и проведение мероприятий** (3.1 – 3.8 детализируют конкретные службы: экология, волонтёры, квизы, лекции, РИА, SMM, интеграции).
4. **Закрытие проекта** (финансовые и организационные отчёты).
5. **Пост-мероприятия** (аналитика, архивирование данных).

**Документация:** Устав проекта, дорожная карта, паспорт мероприятия, смета, расписание, отчёты.

### 2.6. Карта процессов ДСТИ (образовательные заезды)
Аналогична ДКПП, но с акцентом на **образовательный контент** и **научно-методическое сопровождение**. Включает этапы:
1. Подготовка проекта.
2. Согласование концепции.
3. Утверждение контента.
4. Предварительная подготовка к заезду.
5. Организация заезда (заселение, обучение, досуг).
6. Закрытие.
7. Пост-заезд (обратная связь, отчётность).

### 2.7. Карта процессов образовательные заезды (обобщающая)
Структура идентична ДСТИ (этапы 1–7). Включает **ответственные подразделения** по каждому шагу (например, «Служба методологического сопровождения», «Служба работы с партнёрами», «Служба заботы»).

### 2.8. Организация событийного мероприятия (блок-схема)
**Главная схема (верхний уровень):**
- Разработка программы → Защита программы/согласование паспорта → Формирование сметы → Защита сметы.
- **После защиты сметы — три параллельных направления:**
  1. Работа с партнёрами (спонсоры, подрядчики).
  2. Работа со службами (внутреннее взаимодействие).
  3. Работа с договорами.
- **Ветка «Работа с договорами» имеет развилку:**
  - **Сумма <= 1 млн руб.:** Подача заявки на заключение договора ГПХ → Согласование заявки → Заключение договора ГПХ.
  - **Сумма > 1 млн руб.:** Написание ТЗ для договора >1 млн → Согласование заявки → Контрактация → Заключение договора (вероятно, через тендерную процедуру).
- **Слияние трёх направлений** → Реализация мероприятия → Подготовка отчётной документации → Конец.

### 2.9. Процесс подбора сотрудника
Очень длинная схема (35+ шагов). **Роли:** HR-партнёр, Руководитель подразделения (РП), СБ, рекрутер, кадровое делопроизводство, IT-администратор, компенсации и льготы.

**Ключевые этапы:**
1. **Заявка на подбор** (РП формирует, HR согласовывает).
2. **Поиск кандидата** (внутренний/внешний, платный/бесплатный).
3. **Отбор:** первичное интервью (рекрутер), тестовое задание.
4. **Интервью с РП, затем с HR, затем с руководителем более высокого уровня.**
5. **Проверка СБ** (всегда).
6. **Оффер** (согласование ЗП, KPI, даты выхода).
7. **Кадровое оформление** (трудовой договор, приём на работу).
8. **Выдача ноутбука и доступов** (IT) → Онбординг.

### 2.10. Процесс релокации сотрудника
**Три зоны:** КЭДО (Кадровый ЭДО), 1С, Релокация (отдельный блок).

**Этапы:**
1. Потребность в релокации (возникает у руководителя).
2. Формирование СЗ (Служебной записки) в КЭДО.
3. Согласование кадров (проверка наличия ТД/ГПХ).
4. Загрузка паспорта и билета в КЭДО.
5. Переход СЗ в 1С в Департамент сервисного сопровождения.
6. Определение категории релокации (жильё, транспорт, сопровождение).
7. **Две ветки:**
   - **Вариант А (полное сопровождение):** Оформление трансфера → Прибытие релокатора → Обеспечение жилья согласно категории.
   - **Вариант Б (самостоятельный съём):** Самостоятельный съём жилья.
8. **Ветка аренды жилья** (если съём):
   - Предоставление договора аренды → Проверка СБ → Проверка Бухгалтерии → Предоставление ежемесячной компенсации.
9. **Финал** (после выполнения всех действий).

### 2.11. Служба протокольного сопровождения — взаимодействие
**Роли:** Заказчик (инициатор мероприятия), Служба протокольного сопровождения, Служба безопасности, Внешние подрядчики, Служба заботы, Служба связи (IT).

**Ключевые шаги:**
1. Заказчик направляет запрос на протокольное обслуживание.
2. Служба протокольного сопровождения:
   - Запрашивает паспортные данные участников.
   - Формирует список на проход.
   - Передаёт данные в Службу безопасности.
3. СБ проверяет список на соответствие требованиям (санкционные риски, благонадёжность):
   - Если НЕТ → возврат на доработку.
   - Если ДА → Утверждение списка.
4. **Параллельно:** Организация встречи (трансфер, визы, протокольные мероприятия).
5. **Взаимодействие со Службой заботы:** Служба протокола передаёт уточнённый список заезда, Служба заботы организует проживание и заселение.
6. **Контрольная точка «Всё готово» → Начало мероприятия.**
7. Пост-мероприятие: Фиксирование результатов в 1С, закрытие контактов с участниками.

### 2.12. Схема реализации программы (обобщающая)
**Очень большая схема** (вероятно, объединяет все блоки). Содержит:
- Разработка концепции программы.
- Согласование этапов: предзащита, защита.
- Распределение ответственных по блокам: деловая программа, культурная программа, спортивная программа, образовательная программа, волонтёрская программа.
- Технические подготовки (площадки, аудио, видео, свет, трансляции).
- **Интеграция со службами:** забота, протокол, безопасность, партнёры, финансы.
- Реализация (выход в «поле»).
- **Пост-программа:** сбор обратной связи, подготовка презентации итогов, закрытие договоров.

## 3. Ключевые точки принятия решений и пороги

| Условие | Процесс | Решение |
|---------|---------|---------|
| **Сумма договора > 1 млн руб.** | Организация мероприятия | Переход на усложнённую ветку: написание ТЗ, тендерная процедура, отдельный контроль |
| **Проверка заявки СБ** | Заявка на участие (Служба заботы) | Остановка процесса при выявлении рисков |
