## AI

### Функционально-технические требования  
---

#### **1. Общие положения**  
- **Цель**: Автоматизация расчета себестоимости услуг перевозки в разрезе отправлений, маршрутов и ресурсов.  
- **Объект автоматизации**: Подсистема на базе 1С:ERP 2.5, интегрированная с модулями "Производство", "Склад и доставка", "Управление автотранспортом".  
- **Методология расчета**:  
  - Операционно-ориентированный метод (себестоимость отправления = сумма затрат по операциям).  
  - Распределение прямых затрат (труд, материалы, транспорт) и накладных расходов (ОПР, АУР).  

#### **2. Функциональные требования**  
- **Учет ресурсов**:  
  - Трудовые (время сотрудников), материальные (бланки, упаковка), транспортные (пробег, расходы на ТС), услуги сторонних организаций.  
  - Интеграция с модулями "Кадры", "Зарплата", "Управление автотранспортом".  
- **Аналитика**:  
  - В разрезе: виды услуг, проекты, контрагенты, договоры (ГОЗ, ГК), маршруты, транспортные средства, статьи затрат.  
- **Отчетность**:  
  - Формирование калькуляций, детализация до первичных данных, экспорт в форматы XLSX, PDF, XML.  
- **Производительность**:  
  - Время выполнения расчетов ≤ 1.5 сек в пиковый момент.  
  - Оптимизация через кэширование и предварительные расчеты.  

#### **3. Технические требования**  
- **Интеграция**:  
  - Обмен данными с внешними системами через REST API или COM-соединения.  
  - Синхронизация с BI-системами для сложных аналитических расчетов.  
- **Безопасность**:  
  - Ролевая модель доступа (администратор, бухгалтер, менеджер по логистике).  
  - Аудит действий пользователей (логирование операций, авторизаций).  
- **Надежность**:  
  - Ежедневное резервное копирование. Восстановление данных ≤ 24 часов.  

---

### Архитектурное решение  
- **Стек технологий**: 1С:ERP 2.5 + доработки на встроенном языке.  
- **Компоненты**:  
  - **Справочники**: "Маршруты", "Операции", "Ресурсы".  
  - **Документы**: "Заявка на перевозку", "Учет затрат по операции".  
  - **Регистры**: "Затраты по отправлениям", "Распределение накладных расходов".  
- **Интеграция**:  
  - С модулем "Управление автотранспортом" для учета пробега и расходов на ТС.  
  - С BI-платформой (например, Power BI) для аналитики больших объемов данных.

### Риски интегратора и управление ими  
1. **Недостаточная детализация требований к интеграции**:  
   - **Решение**: Включить в ТЗ раздел "Интеграционные спецификации" с описанием форматов данных и протоколов.  

2. **Сложность учета всех видов услуг**:  
   - **Решение**: Добавить в ТЗ приложение с примерами расчета для каждого типа перевозки (например, "Опасные грузы", "Денежная наличность").  

3. **Низкая производительность расчетов**:  
   - **Решение**: Включить требование к нагрузочному тестированию на этапе предварительных испытаний.  

4. **Конфликты с существующими доработками**:  
   - **Решение**: Провести аудит текущей конфигурации 1С:ERP до начала работ.

### Ошибки и неточности в ТЗ  
1. **Отсутствие детализации ролей**: В разделе 4.4 не указаны конкретные роли (например, "Администратор", "Бухгалтер").  
2. **Неясность по нетиповым модулям**: В разделе 3.4 упоминается "Производство (не типовая)", но нет описания ее функционала.  
3. **Противоречие в производительности**: Требование ≤ 1.5 сек не подкреплено анализом объемов данных.

### Рекомендуемые изменения в ТЗ  
1. **Добавить раздел "Интеграционные спецификации"**:

2. **Уточнить методику расчета для каждого вида услуг**:  
   - Включить примеры расчета в Приложение 1 и 2.

   - Определить права доступа для каждой роли (например, "Менеджер по логистике" может только просматривать отчеты).
