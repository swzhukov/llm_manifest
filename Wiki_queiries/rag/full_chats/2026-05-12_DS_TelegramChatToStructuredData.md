## user

Ты — старший Python-разработчик. Среда исполнения: Google Colab (файловая система временная). Использовать только стандартную библиотеку Python и pandas (предустановлены в Colab). Код должен быть выполнен в одной ячейке без интерактивных запросов, без использования pip install. Ответ — исключительно готовый код, без пояснений. Задача: разработать скрипт, превращающий экспорт чата/канала Telegram Desktop в единый структурированный файл, пригодный для передачи нейросети (например, для последующего ответа на вопросы по содержанию чата). Выходной файл должен быть максимально чистым: вся информация извлечена, приведена к плоской таблице, типы данных строгие, отсутствуют пропуски в ключевых полях. Входные данные: директория, заданная переменной INPUT_DIR, содержит один или несколько файлов result*.json (стандартный экспорт Telegram Desktop). Каждый JSON-файл имеет верхний уровень — объект с обязательным ключом "messages", значением которого является массив сообщений. Пример элемента: {"id": 123, "type": "message", "date": "2023-01-15T12:34:56", "date_unixtime": "1673781296", "from": "Иван", "from_id": "user987654321", "text": "Привет", "edited": "2023-01-15T12:40:00", "edited_unixtime": "1673781600", "photo": "photo_1@15-01-2023_12-34-56.jpg", "file": "(File not included…)", "media_type": "sticker", "reply_to_message_id": 120, "forwarded_from": "Канал", "pinned": true}. Поле text может быть строкой или списком элементов (строка или объект {"type":"…", "text":"…"}) — во втором случае нужно склеить все фрагменты последовательно без разделителей, чтобы получить чистый текст. Медиавложения: реальное имя файла хранится в одном из полей photo, video, document, sticker, animation, audio, voice_message (приоритет проверки в этом порядке). Поле file содержит служебное предупреждение, его игнорировать. Если присутствует поле media_type, использовать его значение как тип медиа (приоритет над автоопределением). При отсутствии вложений media_type и media_file — пустые строки. Системные сообщения (type ≠ "message") могут не иметь поля id — такие записи пропускать. Поля from и from_id могут отсутствовать; тогда отправитель "Unknown", идентификатор "". Временные метки: date (ISO), date_unixtime (строка с Unix-временем), аналогично edited/edited_unixtime. Конфигурационные переменные в начале скрипта: INPUT_DIR = "/content/export" (путь к папке с экспортом), OUTPUT_FORMAT = "csv" (или "json"), OUTPUT_FILE = "/content/chat_export.csv". Требования к обработке: найти все result*.json в INPUT_DIR, загрузить, объединить списки messages. При ошибке чтения вывести предупреждение и продолжить. Отбросить записи без id, зафиксировать количество. Создать DataFrame со столбцами: id (Int64), timestamp (datetime64[ns] из date_unixtime или date), sender (str, "Unknown" если нет), sender_id (str, "" если нет), text_clean (str — очищенный текст), text_raw (str — исходное значение text), media_type (str), media_file (str), is_reply (bool, True если reply_to_message_id не null), reply_to_msg_id (Int64, NaN если нет), is_forwarded (bool), forwarded_from (str, "" если нет), is_edited (bool), edit_timestamp (datetime64[ns] из edited_unixtime или edited, иначе NaT), is_pinned (bool). Сортировать строго по timestamp, затем по id. В столбцах text_clean, id, timestamp пропусков не допускается, остальные заполняются пустыми строками, False, NaT/NaN по смыслу. Логирование: вывести количество обработанных файлов, строк до фильтрации, отброшенных, итоговых строк. Экспорт: для CSV encoding='utf-8-sig', index=False; для JSON orient='records', force_ascii=False. После сохранения скачать файл через google.colab.files.download, обернув в try/except ImportError для совместимости вне Colab. Комментарии в коде — краткие, на русском. Итоговый CSV/JSON должен быть полностью готов для загрузки в аналитическую среду или нейросеть: каждая строка — одно сообщение, плоская структура, чистый текст, строгие типы.
