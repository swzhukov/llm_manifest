## ЦЕЛЬ СИСТЕМЫ

Создать персонального инвестиционного агента на базе n8n,
который обеспечивает:
1. Учёт всех инвестиционных сделок через Telegram (текст/голос)
2. Ежемесячную инвестиционную дисциплину (план + напоминание)
3. Регулярный отчёт по портфелю с P&L и рекомендациями
4. Еженедельный дайджест инвестиционных сигналов из
   Telegram-каналов и YouTube

### Лист 2: ПОРТФЕЛЬ
Колонки: инструмент | тикер | блок | кол-во |

  ">="&DATE(YEAR(TODAY()),1,1))]
иис_вычет_потенциал=[=иис_взнос_год_факт*0.13]

## WORKFLOW 1: УЧЁТ СДЕЛКИ

Триггер: Telegram On Message (текст или голос)
Шаги:
1. Switch: тип сообщения (voice / text)
2. Если voice: HTTP POST → Whisper API → текст
3. YandexGPT (агент):
   Системный промпт:
   «Ты парсер инвестиционных сделок. Из входящего
   текста извлеки JSON строго в формате:
   {"тип":"покупка|продажа|пополнение_иис|купон|
   дивиденд|прочее","блок":"стабильный|рост|
   рискованный","инструмент":"название",
   "тикер":"тикер или null","количество":N,
   "цена":N,"сумма":N,"брокер":"название",
   "счёт":"ИИС|брокерский|другой"}.
   Если сумма не указана — вычисли как кол-во×цену.
   Если данных не хватает — верни
   {"status":"need_more","question":"..."}»
4. IF: status == need_more → Telegram: задать вопрос
   ELSE → Google Sheets: append в СДЕЛКИ
5. Telegram: подтверждение с форматированием

## WORKFLOW 2: ЕЖЕМЕСЯЧНЫЙ ПЛАН

Триггер: Cron (1-е число каждого месяца, 09:00 MSK)
+ команда /plan
Шаги:
1. Google Sheets: читаем ПАРАМЕТРЫ и ПОРТФЕЛЬ
2. Рассчитать: текущие доли по блокам
3. YandexGPT:
   «Бюджет: {сумма_мес}₽. Текущие доли:
   стабильный {X}%, рост {Y}%, рискованный {Z}%.
   Целевые: 40/35/25. Рассчитай план на месяц:
   сколько рублей в каждый блок. Отдельно укажи
   статус ИИС: внесено {факт} из {план},
   осталось {остаток}, потенциальный вычет {вычет}₽»
4. Telegram: отправить план

## WORKFLOW 3: ОТЧЁТ ПОРТФЕЛЯ

Триггер: команда /report в Telegram
Шаги:
1. Google Sheets: читаем ПОРТФЕЛЬ
2. Loop по позициям с тикером:
   HTTP GET: https://iss.moex.com/iss/engines/stock/
   markets/shares/securities/{тикер}.json
   Парсим: LAST (последняя цена)
   Обновляем текущую_цену в ПОРТФЕЛЬ
3. Рассчитать P&L, доли
4. YandexGPT (надсмотрщик):
   «Проверь данные. Если доля блока отклоняется
   от целевой более чем на 5% — дай краткую
   рекомендацию (1-2 предложения).»
5. Telegram: форматированный отчёт

Шаги (по схеме Зуевича: n8n + YouTube RSS):
1. Google Sheets: читаем YOUTUBE_КАНАЛЫ
2. Loop по каналам:
   a. RSS Feed Node:
      URL: https://www.youtube.com/feeds/videos.xml
           ?channel_id={channel_id}
   b. Code Node (JS):
      - Фильтр: НЕ Shorts (duration > 3 min)
      - Фильтр: опубликовано не ранее 7 дней назад
      - Извлечь: videoId, title, publishedAt

      subtitle URL → транскрипт первых 500 слов

4. YandexGPT: извлечь инвест-сигналы из описаний/тегов

## PIX RPA (ОПЦИОНАЛЬНО, НЕ MVP)

Назначение: автоматический экспорт данных из
приложения Альфа-Инвестиций (нет публичного API).
Применение: обновление текущей стоимости позиций
в доверительном управлении (ИИС «Точки Роста»).
Триггер: раз в месяц (перед формированием отчёта).
Действие: открыть Альфа-Инвестиции → скринировать
экраны → передать данные через webhook в n8n →
обновить лист ПОРТФЕЛЬ.

## MOEX ISS API: СПРАВКА

Базовый URL для получения последней цены:
https://iss.moex.com/iss/engines/{engine}/markets/{market}/
  securities/{тикер}.json?iss.meta=off&iss.only=marketdata

Для акций: engine=stock, market=shares
Для облигаций: engine=stock, market=bonds
Для БПИФ: engine=stock, market=shares
Поле с ценой: LAST (последняя сделка)
Авторизация: не требуется
Формат ответа: JSON
