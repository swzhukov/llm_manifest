## AI

Ты, скорее всего, столкнулся с ситуацией, когда поведение AI-агента описывают в обычном `.md` файле (Markdown). Да, это очень распространённая практика, но сам агент — **не просто md-файл**. Давай разберёмся, что к чему.

### Почему говорят «это md-файл»?
Во многих современных инструментах логику агента действительно задают текстом в Markdown:

- **Custom GPTs (OpenAI)** — инструкцию (system prompt) пишут прямо в поле, и её часто хранят как `instructions.md`.
- **Open Interpreter** — можно создать профиль `profile.md`, где описана роль, правила, доступ к файлам.
- **Cline / Continue (IDE-агенты)** — системный промпт может лежать в `.clinerules` или `.cursorrules` в формате md.
- **LangChain / CrewAI** — промпт для агента часто выносят в отдельный `agent_prompt.md` и загружают в скрипт.

Но сам `md` — это лишь **текстовый блок инструкций**, который подаётся на вход языковой модели. Без кода, инструментов и цикла выполнения он ничего не делает.

### Что такое AI-агент на самом деле
AI-агент — это программа, которая:
1. Получает задачу от пользователя.
2. Думает (LLM генерирует план или действие).
3. Выполняет действие (запускает код, ищет в интернете, вызывает API).
4. Анализирует результат и решает, достигнута ли цель.
5. Если нет — возвращается к шагу 2.

Таким образом, агент = **LLM + промпт (md) + инструменты + цикл**.

### Как создать простого агента, где промпт лежит в md-файле
Покажу на Python с OpenAI API (можно заменить на любой совместимый). Будем использовать md-файл для системного промпта.

**1. `system.md`** (инструкция агента)
```markdown
Ты — полезный ассистент, который умеет считать и пользоваться поиском.
Всегда объясняй свои действия кратко, а потом выдавай финальный ответ.
Если нужно что-то посчитать — используй инструмент calculator.
Если нужно найти актуальную информацию — используй инструмент search.
```

**2. `agent.py`** (код агента с циклом)
```python
import openai
import json

# Читаем промпт из md-файла
with open("system.md", "r", encoding="utf-8") as f:
    system_prompt = f.read()

# Доступные инструменты (упрощённо)
def calculator(expression):
    return str(eval(expression))  # в реальности — безопасный eval

def search(query):
    # заглушка, в реальности вызов поискового API
    return f"Результаты поиска по запросу: {query}"

tools = {
    "calculator": calculator,
    "search": search
}

# Описание инструментов для модели
tools_schema = [
    {
        "type": "function",
        "function": {
            "name": "calculator",
            "description": "Вычисляет математическое выражение",
            "parameters": {
                "type": "object",
                "properties": {
                    "expression": {"type": "string", "description": "Выражение, например 2+2"}
                },
                "required": ["expression"]
            }
        }
    },
    {
        "type": "function",
        "function": {
            "name": "search",
            "description": "Ищет информацию в интернете",
            "parameters": {
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "Поисковый запрос"}
                },
                "required": ["query"]
            }
        }
    }
]

client = openai.OpenAI()

def run_agent(user_message):
    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": user_message}
    ]

    while True:
        response = client.chat.completions.create(
            model="gpt-4o",
            messages=messages,
            tools=tools_schema,
            tool_choice="auto"
        )
        msg = response.choices[0].message

        # Если модель хочет вызвать инструмент
        if msg.tool_calls:
            messages.append(msg)  # добавляем ответ модели с tool_calls
            for tool_call in msg.tool_calls:
                func_name = tool_call.function.name
                args = json.loads(tool_call.function.arguments)
                result = tools[func_name](**args)
                # Добавляем результат как сообщение от инструмента
                messages.append({
                    "role": "tool",
                    "tool_call_id": tool_call.id,
                    "content": result
                })
        else:
            # Финальный ответ пользователю
            return msg.content

# Пример запуска
print(run_agent("Сколько будет курс доллара к рублю на сегодня и умножь его на 100"))
```

Это уже полноценный агент, а `system.md` — только его «характер».

### Более простой путь: готовые фреймворки
Если не хочешь писать с нуля, возьми:
- **LangChain** / **LangGraph** — там агент определяется кодом, но промпт можно хранить в md.
- **AutoGen** (Microsoft) — опять же, промпт выносится отдельно.
- **CrewAI** — роли агентов задаются прямо в Python, но тексты ролей часто пишут в Markdown для читаемости.
- **OpenAI Custom GPTs** или **Assistants API** — там и есть чисто `md`-конфигурация (инструкция + описание tools в UI/files), и никакого кода писать не надо (но это только в экосистеме OpenAI).

### Резюме
`md-файл` — это лишь удобная упаковка для системного промпта. Настоящий AI-агент живёт в коде, который умеет вызывать инструменты и вести диалог с LLM. Так что да, «агент как md-файл» — сильное упрощение, но очень полезное для быстрого прототипирования.

Если хочешь попробовать прям «создать агента из md» без программирования, посмотри в сторону **OpenAI GPTs** или **Assistants** (через Playground). Там действительно можно загрузить только инструкцию в Markdown, подключить инструменты и получить работающего агента.

### Как этим пользоваться

1. Открываешь веб-версию DeepSeek Chat.
2. Копируешь **весь блок выше** (от «Ты — AI-агент…» до «…действуй строго по этой инструкции.») и вставляешь в поле ввода.
3. После этого, **в том же сообщении**, пишешь свой реальный запрос, например:  
   *«Создай workflow, который при получении вебхука от amoCRM создаёт задачу в Google Таблице и присылает уведомление в Telegram»*.
4. Отправляешь — модель будет держаться роли агента и выдавать JSON+инструкции в нужном формате.

**Важно:** при старте каждого нового диалога (после перезагрузки страницы или сброса контекста) нужно заново вставить этот блок, потому что DeepSeek Chat не сохраняет системную инструкцию между сессиями.

### Почему это работает как md-файл агента

По сути, ты вручную делаешь то, что описано в моём прошлом ответе:  
- **md-файл** = этот текстовый блок с ролью и правилами,  
- **цикл агента** здесь обеспечиваешь ты сам, дочитывая ответ, при необходимости прося уточнения, а потом применяя JSON в n8n.  
Глубокая кастомизация не требуется — получился легковесный специализированный ассистент.

Если в будущем DeepSeek добавит «Проекты» или «Кастомные GPT», ты сможешь сохранить этот промпт там. А пока пользуйся копипастом — это реально улучшит качество ответов именно под твою среду.
