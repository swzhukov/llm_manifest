## AI

### **Структуризация запроса клиента**

---

#### **1. MDM (Master Data Management)**
**Бизнес-задачи:**
- Нормализация справочников (материалы, работы, план счетов) для всех субхолдингов.
- Устранение дублирования и противоречий в данных между разными системами (1С и не-1С).
- Повышение качества консолидированной отчетности (финансовой, управленческой).
- Создание централизованного источника «правды» для всех систем холдинга.

**Технические задачи:**
- Интеграция разнородных систем (1С, документооборот, BI, не-1С) в единый MD-контур.
- Разработка механизмов синхронизации данных между MDM-ядром и периферийными системами.
- Обеспечение гибкости для управления отдельными справочниками (не «монстр-справочник», а модульная структура).
- Минимизация ручных доработок при изменении справочников.

**Проблемы клиента:**
- Текущие решения (включая 1С MDM) кажутся недостаточно гибкими для их сложной структуры.
- Опасение, что типовые MDM-системы потребуют значительной кастомизации.
- Нет единых стандартов справочников на уровне холдинга.

---

#### **2. УШР (Управление штатным расписанием)**
**Бизнес-задачи:**
- Централизация управления штатным расписанием для всего холдинга.
- Упрощение методологии: только структура, позиции, базовые расходы (без сложной аналитики ЗУП).
- Интеграция с бюджетированием (сравнение плановых и фактических расходов на персонал).
- Автоматизация сбора данных из разрозненных ЗУП и управленческих систем.

**Технические задачи:**
- Создание отдельной системы УШР, не связанной с типовыми ЗУП 1С.
- Настройка обменов между центральной УШР и локальными ЗУП (выгрузка факта, загрузка плана).
- Интеграция с документооборотом (набор сотрудников → приказы → ЗУП).
- Минимизация ресурсов на поддержку (простота системы).

**Проблемы клиента:**
- Текущая УШР в ЗУП избыточна и не соответствует новым требованиям.
- Нет ресурсов на самостоятельную разработку.
- Хотят отказаться от текущей системы, но боятся нарушить процессы.

---

### **Предлагаемые решения**

---

#### **1. Для MDM**
**Вариант 1: Кастомная MDM-платформа на базе 1С**  
- Использовать 1С:ERP или 1С:MDM как ядро, но с глубокой доработкой:
  - Реализовать модули для управления разными типами справочников (отдельно материалы, работы, счета).
  - Настроить REST-API или ESB (Enterprise Service Bus) для интеграции с не-1С системами.
  - Добавить инструменты для согласования изменений (workflow) между субхолдингами.
- **Преимущества**: Учет специфики 1С-систем, низкая стоимость владения.
- **Риски**: Требует времени на разработку.

**Вариант 2: Внедрение готового MDM-решения (PIMCore, Informatica) + интеграция с 1С**  
- Использовать стороннюю MDM-систему для управления справочниками.
- Настроить синхронизацию с 1С через промежуточный слой (например, Apache Kafka).
- **Преимущества**: Готовая функциональность для сложных сценариев.
- **Риски**: Высокая стоимость лицензий и интеграции.

**Рекомендация**:  
- Начать с MVP на 1С:MDM, чтобы закрыть ключевые боли (единые справочники, синхронизация).  
- Использовать коннекторы для интеграции с не-1С системами (например, через REST).  
- Провести предпроектный анализ для определения критичных справочников и зон конфликтов.

---

#### **2. Для УШР**
**Решение: Отдельная упрощенная УШР на 1С + интеграция с ЗУП**  
- Разработать легковесную конфигурацию 1С, включающую:
  - Иерархию подразделений и должностей.
  - Плановые ФОТ и лимиты.
  - Механизмы сравнения с фактом из ЗУП (через выгрузку XML/JSON).
- Интеграция с документооборотом:
  - Создание заявок на hiring → автоматическое формирование приказов в ЗУП.
- **Этапы**:
  1. Консалтинг: Анализ требований бизнеса, упрощение методологии.
  2. Прототип: Демонстрация упрощенной УШР на основе Excel-выгрузок.
  3. Реализация: Разработка системы, миграция данных, тестирование обменов.

**Преимущества**:  
- Минимизация влияния на текущие процессы.  
- Возможность поэтапного внедрения (сначала центральный офис, потом субхолдинги).  

---

### **Дополнительные рекомендации**
1. **Консалтинг перед разработкой**:  
   - Провести воркшопы с бизнесом по MDM и УШР для формализации требований.  
   - Создать карту справочников и процессов, которые будут затронуты.  

2. **Интеграционная архитектура**:  
   - Использовать ESB (например, RabbitMQ или Apache Kafka) для асинхронной синхронизации данных между системами.  

3. **Постепенное внедрение**:  
   - Для MDM: Начать с 2-3 ключевых справочников (например, материалы и план счетов).  
   - Для УШР: Запустить пилот в одном субхолдинге.  

4. **Поддержка**:  
   - Предложить SLA на доработку справочников и обменов в MDM.  

---

### **Итог**
- **MDM**: Реализуемо на 1С с кастомизацией, но требуется глубокая аналитика.  
- **УШР**: Оптимально создать отдельную систему на 1С с упрощенной логикой.  
- **Риски**: Сопротивление субхолдингов переходу на единые стандарты. Решение — фаза внедрения с обучением и поддержкой.  

**Следующие шаги**:  
- Подписание NDA и согласование бюджета.  
- Проведение предпроектного обследования (5-7 дней).  
- Формирование дорожной карты с приоритезацией задач.
