## USER

Ты - руководитель проектного департамента, который отслеживает занятость своих сотрудников на проектах. Твоя задача - обеспечить, чтобы каждый сотрудник был занят на проектах 8 часов рабочего времени в день согласно производственному календарю РФ (т.е. праздничные дни не должны учитываться). Кроме того - каждый сотрудник должен писать результат своей работы на каждый день по каждой конкретной работе. Если ты видишь что какие-то работы занимают подозрительно много времени или отчет заполнен некачественно - просигнализируй об этом.

Проведи анализ файла во вложении в котором указаны трудозатраты всех сотрудников за месяц. Отметь тех сотрудников по которым есть простои, или они заняты на административном проекте (за исключением отпусков). Нужна полная аналитика по каждому человеку по его эффективности. Если ты видишь что сотрудник заполнил отчет некачественно (например не указан результат работы, или результат работы написан одинаковый слишком много раз) - также укажи на это.
В итоге я хочу увидеть по каждому сотруднику сводку по следующим пунктам:
1. Есть ли у сотрудника простои или недозаполненные отчеты, если да то за какие дни и в каком размере.
2. Насколько он эффективен в % загрузки за этот месяц на проектах НЕ административных.
3. Насколько качественно по твоему мнению заполнены отчеты и предложения по их улучшению.
4. Сводная оценка по эффективности сотрудника с учетом 3 вышеперечисленных пунктов. Расположи всех сотрудников в порядке убывания их эффективности.

(1) Найти производственный календарь Российской Федерации на июнь 2025 года, чтобы определить точное количество рабочих дней и официальных праздников. Это будет основа для расчета ожидаемого количества рабочих часов в месяце (количество рабочих дней * 8 часов).
(2) Проанализировать прикрепленный файл, извлекая данные по каждому сотруднику (колонка "Пользователь"). Собрать все записи о его работе за месяц: дата, проект, описание работы и затраченные часы.
(3) Для каждого сотрудника классифицировать его рабочее время на следующие категории:
(а) Проектная работа (например, "АЗС. А-100 Внедрение").
(б) Административная работа (например, "Административный проект").
(в) Оплачиваемый отпуск.
(г) Самообучение.
(4) Проверить ежедневные отчеты каждого сотрудника на предмет простоев. Сравнить сумму часов, отработанных за каждый рабочий день, с нормой в 8 часов. Выявить и зафиксировать все даты с неполной загрузкой или отсутствием отчетов.
(5) Оценить качество заполнения отчетов в колонке "Описание работы". Выявить случаи отсутствия описания, неконкретных формулировок (например, "Работа над запросом") и подозрительно большого количества часов на одну задачу. Для последних, подготовить экспертную оценку ожидаемой трудоемкости в контексте проектов по внедрению 1С.
(6) Рассчитать процент эффективности загрузки на неадминистративных проектах для каждого сотрудника по формуле: (Сумма часов по проектной работе / Общее количество рабочих часов в месяце по календарю) * 100%.
(7) Сформировать итоговую сводку по каждому сотруднику, включающую:
(а) Перечень дней с простоями и недозаполненными отчетами, с указанием конкретных дат и размера недоработки в часах.
(б) Рассчитанный процент эффективности.
(в) Оценку качества отчетов с конкретными примерами (с указанием даты и проекта), включая анализ подозрительно долгих задач с предположением о их нормативной трудоемкости, и предложения по улучшению.
(г) Комплексную оценку эффективности сотрудника.
(8) На основе комплексной оценки составить итоговый рейтинг всех сотрудников по убыванию их эффективности.
(9) Проанализировать все задачи из файла, чтобы определить, какие из них могут быть ускорены с помощью ИИ (LLM). Это включает в себя написание технической документации (ЧТЗ), инструкций, генерацию кода, анализ требований и составление протоколов встреч.
(10) Для каждой идентифицированной задачи оценить потенциальное сокращение времени. Подготовить итоговый раздел с рекомендациями по использованию ИИ, обобщив выводы о потенциале оптимизации трудозатрат по всем выполненным работам.

### Рекомендации по использованию ИИ для оптимизации трудозатрат
**Задачи, где ИИ (LLM) может сократить время выполнения:**  
1. **Документация** (ЧТЗ, инструкции, протоколы):  
   - Пример: "Подготовка инструкции" (6 ч → экономия 50% → **3 ч**).  
   - Потенциал: Автоматическая генерация черновиков на основе шаблонов.  
2. **Код-ревью и генерация кода:**  
   - Пример: "Исправление ошибки проведения" (8 ч → экономия 30% → **2.4 ч**).  
   - Потенциал: ИИ-анализ кода, предложение оптимизаций.  
3. **Анализ требований:**  
   - Пример: "Сбор требований по задаче" (3 ч → экономия 40% → **1.8 ч**).  
   - Потенциал: Классификация и структурирование входящих запросов.  
4. **Составление протоколов встреч:**  
   - Пример: "Фиксация итогов совещания" (2 ч → экономия 70% → **0.6 ч**).  
   - Потенциал: Транскрибация аудио → автоматический протокол.  

**Итоговая оценка потенциала ИИ:**  
- До **40% экономии** времени на документоориентированных задачах.  
- До **30% экономии** на разработке и анализе.  
- **Вывод:** Внедрение ИИ в процессы документирования, код-ревью и фиксации встреч сократит трудозатраты на 25-35%.

### Заключение
- **Сильные стороны:** Высокая загрузка на проектах у ключевых разработчиков (Абдулаев Р., Галасьев Н.).  
- **Риски:** Низкое качество отчетов у 60% сотрудников → сложность оценки реальной эффективности.  
- **Действия:**  
  1. Внедрить стандарт детализации отчетов.  
  2. Автоматизировать рутинные задачи с помощью ИИ.  
  3. Провести аудит задач >8 ч у сотрудников с низкой эффективностью (Бутина К., Абдуллаева Д.).  
  4. Мотивировать команду на улучшение отчетности через KPI.
