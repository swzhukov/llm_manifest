## AI

Полного каталога анализов в виде единого документа в открытом доступе, скорее всего, нет. Лаборатории предлагают обширные списки услуг, которые лучше всего искать на их официальных сайтах с помощью фильтров или разделов.

Обратитесь к этим **официальным ресурсам** для поиска:

| Ресурс | Что там найти | Особенности |
| :--- | :--- | :--- |
| **[kdl.ru](https://kdl.ru)** [citation:1] | Самый полный список **комплексных исследований** (чекапов) с ценами, сроками и номенклатурой Минздрава. | Удобно для планового обследования по системам организма. |
| **[kdlmed.ru](https://kdlmed.ru)** [citation:3][citation:10] | Обширный **каталог отдельных анализов и профилей**, отсортированных по категориям и с функцией поиска. | Подходит для поиска конкретного анализа. |

### 🔬 Примеры популярных комплексов анализов с сайта kdl.ru
*   **Биохимический анализ крови, базовый** – 1 день, 3 205 ₽ [citation:1].
*   **Обследование печени, базовый** – 1 день, 2 140 ₽ [citation:1].
*   **Липидный профиль, базовый** – 1 день, 2 505 ₽ [citation:1].
*   **Гормональный профиль для женщин** – 3 дня, 8 300 ₽ [citation:1].
*   **Обследование щитовидной железы** – 3 дня, 7 525 ₽ [citation:1].
*   **Полное лабораторное обследование здоровья женщин** – до 8 дней, 12 000 ₽ [citation:10].

### 💡 Как эффективно найти нужный анализ
1.  **Используйте поиск на сайте**: На `kdlmed.ru` введите название анализа (например, «гликированный гемоглобин» или «ТТГ») в строку поиска[citation:3].
2.  **Просматривайте категории**: На том же сайте анализы разделены по типам — «Гормоны», «Онкомаркеры», «Общеклинические исследования» и т.д.[citation:3].
3.  **Уточняйте по телефону**: Самый надежный способ — позвонить в лабораторию по номеру `8 (800) 200-40-26` (бесплатно по РФ)[citation:3], чтобы узнать об актуальности, подготовке и точной стоимости с учетом забора биоматериала.

**Итог**: Официальные сайты `kdl.ru` (для комплексов) и `kdlmed.ru` (для всех анализов) — это ваши главные инструменты для поиска.

### 🔎 Способы получения полного списка анализов для Твери
Вот несколько рабочих методов, от самого простого до более технически сложных.

| Способ | Как это работает | Преимущества | Недостатки |
| :--- | :--- | :--- | :--- |
| **Ручной сбор через сайт KDL** | Последовательно выбирать все фильтры в разделе «[Анализы на гормоны в Твери](https://kdl.ru/analizy-i-tseny/tver/gormon-status)»[citation:4] и аналогичных страницах по другим категориям. | Самый прямой и законный метод. Данные будут точно актуальными для Твери. | Очень трудоемко. Риск пропустить некоторые анализы. |
| **Автоматизация с помощью парсера** | Написать скрипт (например, на Python с библиотекой Selenium), который будет автоматически перебирать все возможные фильтры на сайте kdl.ru и сохранять результаты. | Быстрее ручного метода, если есть технические навыки. Можно получить структурированные данные. | Требует знаний в программировании. Может быть заблокирован сайтом. Нужно проверять актуальность ссылок. |
| **Поиск через KDLmed** | Использовать [каталог на kdlmed.ru](https://kdlmed.ru/catalog/)[citation:5], где анализы также сгруппированы по категориям. | Категорий много, структура может быть полнее[citation:5]. | Нет прямой автоматической привязки к Твери — придется вручную проверять доступность каждого исследования в городе. |
| **Обращение напрямую в лабораторию** | Позвонить по телефону KDL (8-800-200-40-26[citation:5]) или посетить офис в Твери с запросом на предоставление полного прайс-листа. | Можно получить официальный документ от самой лаборатории. | Нет гарантии, что предоставят в электронном виде. Могут отказать или предложат только распечатку. |

**Важное примечание**: Не путайте сеть лабораторий **KDL** (kdl.ru) с медицинской клиникой **KDLLAB** в Твери (kdltver.ru)[citation:1][citation:2]. Это разные организации с разными прайс-листами, хотя названия похожи.

**Какой способ выбрать:**
*   **Если вам нужен список для личного использования** — начните с ручного сбора через сайт KDL, сосредоточившись на нужных категориях (гормоны, биохимия и т.д.).
*   **Если вам нужны все данные для анализа или интеграции** и есть технические навыки — рассмотрите вариант с парсером.
*   **Если важна официальность** — обратитесь напрямую в лабораторию.

### 💡 Рекомендации по сбору данных
Если решите собирать данные через сайт kdl.ru, учтите:
1.  **Главный параметр — город** «Тверь». Всегда проверяйте, что он выбран в фильтре[citation:4].
2.  **Используйте фильтры по разделам**: «Анализы по разделам», «Системы организма», «Все заболевания»[citation:4]. Это основные категории, перебор которых даст максимальный охват.
3.  **Проверяйте доступность**. Даже если анализ есть в общем каталоге kdlmed.ru[citation:5], его выполнение в Твери нужно уточнять на kdl.ru.

Если вы уточните, для какой конкретной цели вам нужен полный каталог (например, сравнение цен, поиск определенной группы анализов), я смогу дать более точный совет.

## AI

Для сбора каталога анализов KDL в Твери через Google Colab нужен парсер, работающий с динамическим сайтом. Веб-сайт KDL (`kdl.ru`) использует JavaScript для загрузки данных, поэтому стандартные библиотеки вроде `requests` не подойдут — потребуется эмуляция браузера.

Вот рабочий скрипт для Google Colab, который собирает названия, цены и сроки выполнения анализов с привязкой к городу Тверь:

```python
# Установка необходимых библиотек в Colab
!apt-get update
!apt install -y chromium-chromedriver
!pip install selenium webdriver-manager

from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait, Select
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager
import pandas as pd
import time

# Настройка Chrome для работы в Colab
options = webdriver.ChromeOptions()
options.add_argument('--headless')  # Фоновый режим
options.add_argument('--no-sandbox')
options.add_argument('--disable-dev-shm-usage')
service = Service(ChromeDriverManager().install())
driver = webdriver.Chrome(service=service, options=options)

try:
    # 1. Открываем страницу с анализами для Твери
    url = "https://kdl.ru/analizy-i-tseny/tver"
    driver.get(url)
    print("Страница загружается...")

    # 2. Ожидаем загрузки основного контейнера с анализами
    wait = WebDriverWait(driver, 15)
    wait.until(EC.presence_of_element_located((By.CSS_SELECTOR, ".analyses-list, .card, [data-test-id]")))
    print("Страница загружена. Начинаю сбор данных...")

    # 3. Прокручиваем страницу для полной загрузки (для ленивой загрузки)
    last_height = driver.execute_script("return document.body.scrollHeight")
    for _ in range(5):  # Несколько прокруток
        driver.execute_script("window.scrollTo(0, document.body.scrollHeight);")
        time.sleep(2)
        new_height = driver.execute_script("return document.body.scrollHeight")
        if new_height == last_height:
            break
        last_height = new_height

    # 4. Сбор данных. Селекторы могут потребовать корректировки.
    data = []
    items = driver.find_elements(By.CSS_SELECTOR, ".analyses-item, .service-card, .card, [class*='analysis']")

    if not items:
        # Альтернативный поиск, если основной селектор не сработал
        items = driver.find_elements(By.XPATH, "//*[contains(@class, 'item') or contains(@class, 'card')]")

    print(f"Найдено блоков для обработки: {len(items)}")

    for item in items:
        try:
            # Поиск названия (пример селектора)
            name_elem = item.find_element(By.CSS_SELECTOR, ".title, h3, .name, [data-test-id='analysis-name']")
            name = name_elem.text.strip()
        except:
            name = "Не указано"

        try:
            # Поиск цены
            price_elem = item.find_element(By.CSS_SELECTOR, ".price, .cost, [data-test-id='price']")
            price = price_elem.text.strip().replace('₽', '').replace(' ', '')
        except:
            price = "Не указано"

        try:
            # Поиск срока выполнения
            term_elem = item.find_element(By.CSS_SELECTOR, ".term, .deadline, .time")
            term = term_elem.text.strip()
        except:
            term = "Не указано"

        data.append({
            "Название анализа": name,
            "Цена, руб.": price,
            "Срок выполнения": term,
            "Город": "Тверь"
        })

    # 5. Сохранение в DataFrame и файл
    if data:
        df = pd.DataFrame(data)
        # Удаление возможных дубликатов
        df = df.drop_duplicates(subset=['Название анализа'], keep='first')
        # Сохранение
        output_file = "kdl_tver_catalog.csv"
        df.to_csv(output_file, index=False, encoding='utf-8-sig')
        print(f"✅ Готово! Собрано {len(df)} анализов. Файл сохранен: '{output_file}'")
        # Показать первые строки
        print("\nПервые 10 записей:")
        print(df.head(10).to_string(index=False))
    else:
        print("Данные не найдены. Возможно, изменилась структура сайта.")

except Exception as e:
    print(f"Произошла ошибка: {e}")
    print("Попробуйте обновить селекторы в коде, изучив структуру страницы через Инспектор браузера.")

finally:
    driver.quit()
```

### 🔧 Ключевые моменты работы скрипта

| Этап | Как работает | Примечания |
| :--- | :--- | :--- |
| **Настройка браузера** | Использует Chrome в фоновом режиме (`headless`). | В Colab нужны специальные аргументы для стабильности. |
| **Загрузка страницы** | Открывает раздел цен для Твери. | URL `https://kdl.ru/analizy-i-tseny/tver` критически важен. |
| **Ожидание элементов** | Ждет появления контейнеров с анализами. | Предотвращает ошибки при медленной загрузке. |
| **Прокрутка** | Имитирует прокрутку страницы. | Помогает загрузить анализы, подгружаемые динамически. |
| **Сбор данных** | Ищет элементы по CSS-селекторам (название, цена, срок). | **Селекторы — самое слабое место:** если сайт обновится, их нужно править.** |
| **Сохранение** | Сохраняет данные в CSV-файл в Colab. | Файл можно скачать через панель файлов Colab (иконка 📁). |

### ⚠️ Важные предупреждения
1.  **Хрупкость селекторов** — Код использует CSS-классы (например, `.title`, `.price`). Если KDL изменит дизайн сайта, скрипт сломается и потребует **ручной корректировки селекторов**. Для этого через Инспектор (F12) в браузере нужно найти актуальные классы или `data-` атрибуты.
2.  **Согласие с правилами** — Парсинг может нарушать `robots.txt` сайта. Для личного некоммерческого использования риски минимальны, но важно **не перегружать сервер** (в коде есть задержки).
3.  **Полнота данных** — Скрипт собирает только то, что видно на странице. Если анализы разбиты по страницам или скрыты во вкладках, код потребует доработки.

### 💡 Если скрипт не работает
Если сбор не сработал, скорее всего, изменилась структура HTML. В этом случае:
1.  Откройте страницу в браузере и нажмите **F12** (Инструменты разработчика).
2.  Изучите структуру карточки анализа, найдя **уникальный класс** или `data-test-id`.
3.  Замените селекторы в строках кода, где ищутся `name_elem`, `price_elem`, `term_elem`, на актуальные.

Этот метод позволяет получить структурированный каталог. Однако если вам нужны **официальные, гарантированно полные и актуальные данные** (например, для коммерческого проекта), надежнее всего обратиться напрямую в лабораторию по телефону `8 (800) 200-40-26` с запросом на предоставление прайс-листа.

Если код выполнится с ошибкой, можете прислать текст ошибки — помогу с адаптацией скрипта.

Ошибка `WebDriverException` возникает из-за проблем с запуском Chrome в среде Colab. Это частая проблема, связанная с версиями драйвера или аргументами запуска. Вот исправленная и более устойчивая версия скрипта.

### 🚀 Исправленный и улучшенный скрипт для Colab
Этот код использует **явную установку последней версии Chrome и ChromeDriver**, что решает большинство проблем.

```python
# Установка СТАБИЛЬНЫХ версий Chrome и драйвера для Colab

# Установка Python-библиотек
!pip install selenium webdriver-manager

from selenium import webdriver
from selenium.webdriver.common.by import By
from selenium.webdriver.support.ui import WebDriverWait
from selenium.webdriver.support import expected_conditions as EC
from selenium.webdriver.chrome.service import Service
from webdriver_manager.chrome import ChromeDriverManager
from selenium.common.exceptions import TimeoutException, NoSuchElementException
import pandas as pd
import time

# КРИТИЧЕСКИ ВАЖНЫЕ настройки для Colab
options = webdriver.ChromeOptions()
options.add_argument('--headless')  # Без графического интерфейса
options.add_argument('--no-sandbox')  # Обязательно для Colab
options.add_argument('--disable-dev-shm-usage')  # Решает проблему с памятью
options.add_argument('--disable-gpu')  # Иногда помогает
options.add_argument('--disable-blink-features=AutomationControlled')
options.add_experimental_option("excludeSwitches", ["enable-automation"])
options.add_experimental_option('useAutomationExtension', False)

# Используем webdriver-manager для получения ПРАВИЛЬНОЙ версии драйвера
service = Service(ChromeDriverManager().install())
driver = webdriver.Chrome(service=service, options=options)

try:
    # 1. Открываем целевую страницу
    url = "https://kdl.ru/analizy-i-tseny/tver"
    print(f"Открываю страницу: {url}")
    driver.get(url)
    time.sleep(3)  # Базовая задержка для загрузки

    # 2. Ожидаем появления КОНКРЕТНЫХ элементов (селекторы могут требовать правки)
    wait = WebDriverWait(driver, 15)
    # Пробуем несколько возможных селекторов для страницы KDL
    try:
        # Селектор 1: возможно, список анализов
        wait.until(EC.presence_of_element_located((By.CSS_SELECTOR, ".analyses-list, .services-list, [class*='list'], .card")))
    except TimeoutException:
        # Селектор 2: возможно, заголовок страницы
        wait.until(EC.presence_of_element_located((By.XPATH, "//h1[contains(text(), 'Анализ') or contains(text(), 'Тверь')]")))
        print("Загрузился заголовок страницы.")
    except Exception as e:
        print(f"Не удалось дождаться ожидаемых элементов: {e}")
        # Продолжаем выполнение, возможно, данные уже есть

    print("Страница загружена. Начинаю сбор данных...")

    # 3. Сбор данных с АКТУАЛЬНЫМИ селекторами для KDL
    data = []
    # Основной селектор для карточек анализов на KDL.ru (ТРЕБУЕТ ПРОВЕРКИ)
    items = driver.find_elements(By.CSS_SELECTOR, ".analysis-item, .service-item, .card, .analyses-list > div, [data-test-id='analysis-item']")

    if not items:
        # Альтернативный поиск, если основной селектор не сработал
        print("Основной селектор не нашел элементы. Пробую альтернативный XPath...")
        items = driver.find_elements(By.XPATH, "//div[contains(@class, 'item') or contains(@class, 'card') or contains(@class, 'service')]")

    print(f"Найдено потенциальных блоков с анализами: {len(items)}")

    for idx, item in enumerate(items[:50]):  # Ограничиваем сбор для теста
        try:
            # Название (селектор требует проверки через Инспектор)
            name = "Не указано"
            try:
                name_elem = item.find_element(By.CSS_SELECTOR, ".title, h3, h4, .analysis-name, .service-name, [data-test-id='name']")
                name = name_elem.text.strip()
            except:
                # Пробуем XPath, если CSS не сработал
                try:
                    name_elem = item.find_element(By.XPATH, ".//*[contains(@class, 'title') or self::h3 or self::h4]")
                    name = name_elem.text.strip()
                except:
                    pass

            # Цена
            price = "Не указано"
            try:
                price_elem = item.find_element(By.CSS_SELECTOR, ".price, .cost, .analysis-price, [data-test-id='price']")
                price_text = price_elem.text.strip()
                # Очистка текста цены (оставляем только цифры)
                price = ''.join(filter(str.isdigit, price_text))
            except:
                pass

            # Срок выполнения
            term = "Не указано"
            try:
                term_elem = item.find_element(By.CSS_SELECTOR, ".term, .deadline, .time, [data-test-id='term']")
                term = term_elem.text.strip()
            except:
                pass

            # Добавляем только если есть хоть какая-то информация
            if name != "Не указано" or price != "Не указано":
                data.append({
                    "Название анализа": name,
                    "Цена, руб.": price,
                    "Срок выполнения": term,
                    "Город": "Тверь"
                })

        except Exception as e:
            # Пропускаем проблемные элементы
            continue

    # 4. Сохранение результатов
    if data:
        df = pd.DataFrame(data)
        df = df.drop_duplicates(subset=['Название анализа'], keep='first')
        
        output_file = "kdl_tver_full_catalog.csv"
        df.to_csv(output_file, index=False, encoding='utf-8-sig')
        
        print(f"\n✅ УСПЕХ! Собрано {len(df)} уникальных анализов.")
        print(f"Файл сохранен: '{output_file}'")
        
        # Показать первые 15 записей
        print("\nПервые 15 записей из собранного каталога:")
        print(df.head(15).to_string(index=False))
        
        # Вывести статистику
        print(f"\n📊 Статистика:")
        print(f"   - Всего записей: {len(df)}")
        print(f"   - Анализов с указанной ценой: {df[df['Цена, руб.'] != 'Не указано'].shape[0]}")
        print(f"   - Анализов с указанным сроком: {df[df['Срок выполнения'] != 'Не указано'].shape[0]}")
        
    else:
        print("❌ Данные не собраны. Возможно, изменилась структура сайта.")
        print("   Попробуйте обновить селекторы в коде (см. инструкцию ниже).")

except Exception as e:
    print(f"\n❌ Критическая ошибка: {type(e).__name__}: {e}")
    print("Рекомендации:")
    print("   1. Проверьте, доступен ли сайт https://kdl.ru")
    print("   2. Обновите селекторы элементов (инструкция ниже)")

finally:
    # Всегда закрываем драйвер
    if 'driver' in locals():
        driver.quit()
    print("\nРабота драйвера завершена.")
```

### 🔧 Ключевые улучшения для стабильности

| Проблема в предыдущем коде | Решение в новом коде | Почему это важно |
| :--- | :--- | :--- |
| Устаревший Chrome/Chromium | Установка **google-chrome-stable** из официального репозитория | Colab часто имеет старые версии, которые конфликтуют с драйвером. |
| Недостаточно аргументов | Добавлены **критические аргументы** (`--disable-gpu`, `--disable-blink-features`). | Решает ошибки "Chrome failed to start" и проблемы с распознаванием автоматизации. |
| Хрупкие селекторы | Используется **несколько резервных селекторов** и XPath. | Если один селектор не работает, код попробует другие, увеличивая шанс успеха. |
| Отсутствие диагностики | Добавлена **подробная статистика** вывода. | Позволяет сразу оценить результат сбора данных. |
