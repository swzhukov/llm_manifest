## ЦЕЛЬ СИСТЕМЫ

Создать персонального инвестиционного агента на базе n8n,
который обеспечивает:
1. Учёт всех инвестиционных сделок через Telegram (текст/голос)
2. Ежемесячную инвестиционную дисциплину (план + напоминание)
3. Регулярный отчёт по портфелю с P&L и рекомендациями
4. Еженедельный дайджест инвестиционных сигналов из
   Telegram-каналов и YouTube

## WORKFLOW 1: УЧЁТ СДЕЛКИ

Триггер: Telegram On Message (текст или голос)
Шаги:
1. Switch: тип сообщения (voice / text)
2. Если voice: HTTP POST → Whisper API → текст
3. YandexGPT (агент):
   Системный промпт:
   «Ты парсер инвестиционных сделок. Из входящего
   текста извлеки JSON строго в формате:
   {"тип":"покупка|продажа|пополнение_иис|купон|
   дивиденд|прочее","блок":"стабильный|рост|
   рискованный","инструмент":"название",
   "тикер":"тикер или null","количество":N,
   "цена":N,"сумма":N,"брокер":"название",
   "счёт":"ИИС|брокерский|другой"}.
   Если сумма не указана — вычисли как кол-во×цену.
   Если данных не хватает — верни
   {"status":"need_more","question":"..."}»
4. IF: status == need_more → Telegram: задать вопрос
   ELSE → Google Sheets: append в СДЕЛКИ
5. Telegram: подтверждение с форматированием

## MOEX ISS API: СПРАВКА

Базовый URL для получения последней цены:
https://iss.moex.com/iss/engines/{engine}/markets/{market}/
  securities/{тикер}.json?iss.meta=off&iss.only=marketdata

Для акций: engine=stock, market=shares
Для облигаций: engine=stock, market=bonds
Для БПИФ: engine=stock, market=shares
Поле с ценой: LAST (последняя сделка)
Авторизация: не требуется
Формат ответа: JSON

## AI

Отлично, у вас очень детально проработанное ТЗ. Как персональный инвестиционный агент, я помогу вам превратить эту спецификацию в живые рабочие процессы. Ниже — пошаговая дорожная карта, готовая конфигурация для n8n и важные технические детали.

## 1. Google Sheets — закладываем фундамент

Создайте таблицу с пятью листами точно по вашей структуре. Обратите внимание на формулы в **ПАРАМЕТРЫ**:

- `иис_взнос_год_факт`:  
  `=SUMIFS(СДЕЛКИ!G:G; СДЕЛКИ!D:D; "пополнение_иис"; СДЕЛКИ!A:A; ">="&DATE(YEAR(TODAY());1;1))`
- `иис_вычет_потенциал`:  
  `=иис_взнос_год_факт*0,13`  
  (формат ячейки — рубли)

В листе **YOUTUBE_КАНАЛЫ** сразу заполните столбцы: название, channel_id, rss_url.  
Пример для Smart-Lab:  
`https://www.youtube.com/feeds/videos.xml?channel_id=UCdL1E…`  
(найдите точный ID через сайт канала).

## 2. Общий принцип: YandexGPT и двойная валидация

Вы используете связку **Агент + Надсмотрщик**. В n8n это реализуется двумя последовательными узлами AI Agent (или HTTP Request к YandexGPT) с разными системными промптами.

**Важно:** YandexGPT API ожидает строгий JSON-формат инструкций. Используйте узел **HTTP Request** с методом POST к `https://llm.api.cloud.yandex.net/foundationModels/v1/completion`.  
Аутентификация — IAM-токен (или API-ключ, но IAM надёжнее). Я дам готовые примеры промптов.

## 3. Рабочие процессы n8n (по неделям MVP)

### Неделя 1: Таблицы созданы → переходим к Week 2

### Неделя 2: WF4a — Telegram-дайджест (понедельник, 09:00)

**Триггер:** Schedule (Cron: `0 9 * * 1`)

**Шаги:**

1. **12 HTTP Request узлов** (выполняются параллельно).  
   Каждый забирает RSS канала. Например, для @smartlabnews используйте RSSHub:  
   `https://rsshub.app/telegram/channel/smartlabnews`.  
   (Для ЦБ РФ: `https://rsshub.app/telegram/channel/centralbank_russia`).  
   *Лайфхак:* используйте node "Execute Workflow" для перебора, если не хотите 12 узлов.

2. **Merge** — объединяете все items в один массив.

3. **Code Node (JS)** для фильтрации:
   ```javascript
   const weekAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000);
   return items.filter(item => new Date(item.json.pubDate) > weekAgo);
   ```

4. **YandexGPT (Агент)** — HTTP Request.
   ```json
   {
     "modelUri": "gpt://<folder_id>/yandexgpt/latest",
     "completionOptions": {
       "stream": false,
       "temperature": 0.3,
       "maxTokens": "4000"
     },
     "messages": [
       {
         "role": "system",
         "text": "Ты инвестаналитик. Извлеки из постов ТОЛЬКО сигналы с конкретным инструментом. Верни нумерованный список, каждый пункт: [тег] Источник: сигнал (обоснование). Максимум 10."
       },
       {
         "role": "user",
         "text": "{{$json.posts}}"
       }
     ]
   }
   ```

5. **YandexGPT (Надсмотрщик)** — повторный запрос к модели, но промпт:  
   «Проверь каждый пункт: есть ли конкретный инструмент? Если нет — удали. Оставь только корректные. Верни чистый список.»

6. **Google Sheets Append** — в ДАЙДЖЕСТ_ЛОГ (поля: дата, источник, сигнал, тег, использован=false).

7. **Telegram Send** — отформатированное сообщение.

### Неделя 3: WF1 — Учёт сделки (Telegram)

**Триггер:** Telegram Trigger (на любое текстовое или voice-сообщение)

**Маршрутизация:**
- Узел **Switch**: `item.json.message.voice` → путь «Голос», иначе «Текст».
- **Голос:** HTTP Request к Whisper API (используйте `https://api.openai.com/v1/audio/transcriptions` или self-hosted Whisper).  
  → На выходе текст.
- **Общий путь:** Code Node для очистки текста.

**YandexGPT-парсер (Агент):**
Системный промпт (как в ТЗ):
«Ты парсер сделок. Из текста извлеки JSON: {"тип":"...", ..., "сумма":N}. Если не хватает данных — {"status":"need_more","question":"..."}»

**Условный узел IF:**
- Если `status` = "need_more" → Telegram Send Message (reply to user) с текстом вопроса.
- Иначе → Google Sheets Append в лист **СДЕЛКИ**.

**Подтверждение:**
Telegram Send: «✅ Сделка записана: {тип} {инструмент} на {сумма}₽»

### Неделя 4: WF2 — Ежемесячный план + WF3 — Отчёт

**WF2 (Cron 0 9 1 * * или /plan):**
1. Google Sheets Read из ПАРАМЕТРЫ и ПОРТФЕЛЬ.
2. Code Node: вычисляем текущие доли (текущая_стоимость / общая_стоимость * 100%).
3. YandexGPT: передаём структуру с бюджетом и долями, получаем план распределения.  
   Промпт: «Бюджет: 15000₽. Текущие доли: стаб 38%, рост 40%, риск 22%. Целевые 40/35/25. Предложи, сколько рублей в каждый блок. Также проверь статус ИИС: внесено X из 400000, осталось Y, вычет Z₽.»
4. Telegram с красивым форматированием.

**WF3 (команда /report):**
1. Читаем ПОРТФЕЛЬ.
2. **Loop** по позициям, у которых есть тикер.
   - HTTP Request к MOEX ISS:  
     `https://iss.moex.com/iss/engines/stock/markets/shares/securities/{тикер}.json?iss.meta=off&iss.only=marketdata`
   - Парсим `marketdata.data` → массив, последний столбец `LAST` (текущая цена). Для облигаций `engine=stock&market=bonds`.
3. Code Node: обновляем ячейки в Google Sheets (batch update), пересчитываем P&L и доли.
4. YandexGPT (Надсмотрщик): «Если отклонение от целевой доли >5%, дай рекомендацию в 1-2 предложения. Итоговый отчёт должен содержать позиции, P&L, доли, комментарий.»
5. Telegram-сообщение с таблицей в MarkdownV2 или вручную сверстанное.

## 4. Продвинутые моменты и «фишки»

**MOEX ISS:** Обратите внимание, что поле `LAST` может быть null для неликвидных бумаг. Делайте fallback на `PREVPRICE` или пропускайте позицию с уведомлением.

**YouTube WF4b:** Используйте RSS узел (YouTube feeds). После получения видео вызывайте **YouTube Data API v3** для получения description, tags.  
Whisper для транскрипции: требуется скачивание аудиодорожки (youtube-dl), в n8n можно сделать через Execute Command. В MVP это опционально, пока ограничьтесь анализом названий и описаний.

**Pix RPA:** Когда дойдете до этой стадии, сделаете отдельный workflow, который принимает webhook от Pix RPA (скриншоты в base64) и через YandexGPT Vision извлекает цифры. Пока — ручной ввод.
