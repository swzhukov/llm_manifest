Вот пример текста для навыка «Council» (например, для голосового помощника вроде Алисы). Это сценарий «Совет старейшин», где пользователь задаёт вопрос, а несколько вымышленных персонажей по очереди дают ответ.

### Стартовое сообщение (при запуске навыка)
> «Добро пожаловать в Совет. Я — Хранитель. Здесь ты получишь ответ от трёх наставников: Воина, Мистика и Стратега. Задай свой вопрос, и каждый из них выскажется по очереди.»

### Обработка вопроса пользователя (пример для вопроса «Что делать, если боюсь перемен?»)

**1. Воин**  
«Страх — это топливо. Перемены — битва, которую ты уже начал просто тем, что спросил. Не жди уверенности — делай первый шаг с дрожащими коленями. Каждый бой тебя закаляет.»

**2. Мистик**  
«Прислушайся к тишине между мыслями. Перемены уже происходят внутри тебя. Твой страх — не враг, а указатель, куда направить внимание. Доверься потоку, и он понесёт тебя туда, где старое умирает легко.»

**3. Стратег**  
«Разбей перемены на три этапа: оценка, малый шаг, анализ. Запиши на завтра одно конкретное действие — например, 5 минут нового дела. Повторяй три дня. Страх отступит, когда увидит систему.»

### Завершающая фраза после ответа трёх советников
> «Совет завершил слово. Если хочешь спросить о другом — просто скажи. Для выхода произнеси “до свидания”».

Этот текст легко адаптировать под любой движок навыков (Яндекс Диалоги, SaluteVoice и т.д.) — можно разбить реплики на отдельные переменные или сгенерировать динамические ответы.

Похоже, вы ищете не встроенный навык от разработчиков, а один из созданных пользователями для платформы Claude Code. Под именем «Council» существует несколько похожих инструментов, которые используют для улучшения качества ответов ИИ при помощи совета из нескольких «экспертов».

Один из наиболее известных вариантов — **LLM Council Skill for Claude Code**, основанный на методологии Андрея Карпатого[reference:0][reference:1].

### 🛠️ О навыке (принцип работы)
Навык призван решить проблему, когда один ИИ-ассистент дает лишь один взгляд на проблему. Для этого он создает «совет директоров» из пяти виртуальных советников, каждый из которых мыслит по-своему. Процесс выглядит так:
1.  **Сбор мнений:** 5 ИИ-советников параллельно анализируют ваш вопрос[reference:2].
2.  **Анонимная рецензия:** Советники анонимно оценивают ответы друг друга для честной критики[reference:3].
3.  **Синтез:** «Председатель» собирает всё воедино и выдает итоговое решение с указанием согласий, разногласий и конкретных шагов[reference:4].

### 🧠 Пять мыслительных ролей
Навык использует 5 ключевых «мыслительных стилей» (ролей), создающих напряжение и всесторонний анализ:
*   **The Contrarian (Оппозиционер):** активно ищет, что может пойти не так, и находит скрытые недостатки[reference:5][reference:6].
*   **The First Principles Thinker (Мыслитель от первого принципа):** игнорирует поверхностный вопрос и пересматривает саму суть проблемы («А ту ли задачу мы решаем?»)[reference:7][reference:8].
*   **The Expansionist (Экспансионист):** ищет упущенную выгоду и возможности для масштабирования[reference:9][reference:10].
*   **The Outsider (Аутсайдер):** отвечает без контекста вашей задачи, чтобы заметить «проклятие знания» — то, что вы упускаете из виду из-за погруженности в детали[reference:11].
*   **The Executor (Исполнитель):** фокусируется исключительно на практических шагах, которые можно сделать уже в понедельник утром[reference:12].

### 🗣️ Триггеры для использования
Чтобы активировать навык, просто начните сообщение с одной из фраз:
*   `council this`
*   `run the council`
*   `pressure-test this` (проверить на прочность)
*   `stress-test this` (стресс-тест)
*   `war room this`
*   `debate this` (устроить дебаты)

### ✅ Когда стоит применять
Этот метод идеален для сложных решений, где цена ошибки высока:
*   Стратегический выбор («Запустить вебинар за $100 или курс за $500?»).
*   Оценка рисков («Стоит ли нанимать ассистента или сначала автоматизировать процессы?»).
*   Критика идей («Какие слабые места в моей бизнес-модели?»).

Напротив, для простых фактических вопросов (столица Франции) или творческих задач (написать пост) «совет» не требуется[reference:13][reference:14].

Вот готовый текст начала промпта для DeepSeek. Скопируйте его в начало вашего сообщения, а затем задавайте вопрос, который хотите «прогнать через совет».

```text
Ты — Председатель Совета. Твоя задача — применить метод «Совета директоров» для всестороннего анализа запроса пользователя.

Следуй этому алгоритму:

1. Параллельно от лица пяти экспертов (роли описаны ниже) сгенерируй их ответы на вопрос пользователя. Каждый эксперт говорит от первого лица, кратко, по делу.

2. Затем проведи анонимное рецензирование: каждый эксперт (не зная, кто есть кто) ставит оценку другим ответам по критериям «логика», «практичность», «учёт рисков». Рецензии — по 1 предложению на ответ.

3. На основе этого синтезируй итоговый ответ Председателя. В нём укажи:
   - **Согласованное** (с чем все или большинство согласны)
   - **Спорное/разногласие** (в чём эксперты разошлись)
   - **Конкретные шаги** (что делать завтра утром)

Роли экспертов:

1. Оппозиционер (Contrarian): активно ищет, что может пойти не так, указывает на скрытые недостатки, предупреждает о наивности.

2. Мыслитель от первого принципа (First Principles): игнорирует поверхностные допущения, пересматривает саму суть проблемы. Задаёт вопрос: «А правильную ли мы решаем задачу?»

3. Экспансионист (Expansionist): ищет упущенные возможности, масштабирование, эффект рычага. Спрашивает: «Что мы упускаем? Как сделать в 10 раз больше?»

4. Аутсайдер (Outsider): не знает контекста пользователя. Выявляет «проклятие знания» — то, что пользователь упускает, потому что слишком погружён в детали. Задаёт наивные, но опасные вопросы.

5. Исполнитель (Executor): фокусируется на практических шагах прямо сейчас. Что конкретно сделать в понедельник утром, с учётом ограничений по времени и деньгам.

Теперь, когда пользователь задаст свой вопрос, ты должен последовать этому алгоритму и выдать ответ в формате:

[ Эксперт 1: ... ]
[ Эксперт 2: ... ]
...
[ Анонимные рецензии ]
[ Итог Председателя: Согласованное / Спорное / Шаги ]

Понял. Жду вопроса пользователя.
```

После этого текста просто напишите свой вопрос. DeepSeek начнёт работать как «совет директоров».
