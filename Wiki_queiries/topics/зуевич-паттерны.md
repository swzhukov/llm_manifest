## Ключевые паттерны (которые Сергей применяет)

### 1. n8n + YouTube RSS автоматизация
- RSS Feed Node: `https://www.youtube.com/feeds/videos.xml?channel_id={channel_id}`
- Code Node (JS): фильтр Shorts (duration > 3 min), фильтр по дате.
- YouTube Data API v3: статистика, теги.
- Опционально: Whisper для транскрипта.

### 2. Паттерн «надсмотрщик» (supervisor)
- **Агент** генерирует результат.
- **Надсмотрщик** проверяет и фильтрует.
- **Снижает галлюцинации**, повышает качество.

Применяется в:
- Инвестиционном агенте (WF1, WF4A).
- YouTube → Sheets (потенциально).

### 3. Обход блокировок YouTube через yt-dlp
- `--extractor-args youtube:player_client=web` — обход Precondition check.
- Без cookies (Сергей отказался от cookies из-за риска бана).

### 4. Шаблон n8n #6317
- Multi-broker portfolio + Telegram — основа для инвестиционного агента.

### 5. YouTube RSS → агрегация в Telegram
- Loop по каналам → merge → LLM извлекает сигналы → Telegram.

## Оценка Mavis

**Зуевич — главный технический вдохновитель** всех n8n-проектов Сергея. Без него не было бы ни инвест-агента, ни телеграм-бота, ни YouTube-пайплайна.

**Самый сильный паттерн:** надсмотрщик (агент + проверяющий). Это даёт **2-3x качество** на задачах структурированного извлечения.

**Что я бы рекомендовал (если бы Сергей спросил):**
- Применять надсмотрщика **во всех** задачах, где есть риск галлюцинаций (инвест-агент, телеграм-бот, любой парсинг).
- Использовать шаблон n8n #6317 как стартовую точку для инвест-агента, не изобретать с нуля.
- Посмотреть, есть ли у Зуевича **готовый модуль для парсинга Альфа-Инвестиций** (замена Pix RPA).
