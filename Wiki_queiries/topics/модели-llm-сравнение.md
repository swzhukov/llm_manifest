# Сравнение LLM-моделей (из опыта Сергея)

> **Источники:** `raw_imports/00_DS_Лучшая модель выбора.md`, `raw_imports/01_DS_Дипсик и его трепыхания.md`, `raw_imports/06_DS_Что это и зачем.md`, QW 2026-06-15.

---

## Контекст

Сергей **активно экспериментирует** с разными LLM-клиентами (DeepSeek, Kimi, Qwen, Minimax, Perplexity). Это часть его методологии — давать одну задачу нескольким моделям и сравнивать.

---

## Что использует (по факту в чатах)

| Модель | Где использует |
|--------|----------------|
| **DeepSeek** (V3.1, V3.2, R1) | Основная для большинства задач. Разочарован «жалкими трепыханиями». |
| **Qwen** (мультимодальный) | Для сложных аналитических задач (Кадровые проблемы, Karpathy, продажи). |
| **Kimi** (200k контекста) | Упоминается, но не видно прямого использования. |
| **Minimax** (4M контекста) | Упоминается, но не видно прямого использования. |
| **Perplexity** | Упоминается как «AI-исследователь» (для поиска). |
| **YandexGPT** | В n8n-агентах (продакшн). |
| **YandexGPT (Whisper API)** | Транскрипция голоса в инвест-агенте. |

---

## Что выбирает Сергей (по сценариям)

| Сценарий | Лучшая модель (по Сергею) |
|----------|---------------------------|
| **Большой код (YouTube → Sheets)** | Модель, которая даёт сразу рабочий код (не DeepSeek, проверено) |
| **Сложный аналитический разбор (Кадровые)** | Qwen (лучше DeepSeek) |
| **Промышленная маркировка — стратегия** | DeepSeek (по умолчанию) |
| **Генеалогия — поиск в источниках** | Пока не нашёл подходящую |
| **Структура инвест-агента** | DeepSeek (ТЗ получилось детальное) |
| **Продажи (аудит встречи)** | Qwen (по умолчанию для продаж) |
| **Долголетие (научный разбор)** | DeepSeek |
| **Оптимизация решений по паттернам** | DeepSeek (берёт несколько предыдущих решений) |

---

## Параметры, которые выбирает Сергей

Из чата `00_DS_Лучшая модель выбора.md`:
- **«Точность»** — главный приоритет.
- **«Только веб-интерфейс»** — нет доступа к API.
- Использует **«Глубокое мышление» (R1)** для сложных задач.
- **Длинный контекст** — для больших задач (ТЗ, транскрипты).

## Боль и проблемы с моделями

| Боль | Пример |
|------|--------|
| **LLM задают вопросы, когда данные уже даны** | Метавр: 3 раза пнул, LLM не сделал |
| **LLM не сходят в источники** | Генеалогия: 7 сообщений, ничего не найдено |
| **LLM возвращают поверхностные ответы** | Префабрика АГ: «как-то очень поверхностно ты подошел» |
| **LLM «тревожно» уточняют** | «А вы уверены?», «Может, лучше уточнить?» |
| **DeepSeek «трепыхается»** | «жалкие трепыхания дипсика» (сравнение с Qwen, Kimi) |

---

## Вендорный стек, который Сергей построил (для n8n)

В **продакшн-проектах** (инвест-агент, телеграм-бот) Сергей использует:

- **YandexGPT** — основной LLM-агент и надсмотрщик
- **Whisper API** — для голоса
- **MOEX ISS** — для котировок

**Почему не OpenAI/Anthropic:** видимо, вопрос санкций, локализации, стоимости.

## Решения по выбору модели (что я рекомендую)

### Для Mavis (если Сергей спросит «какую модель выбрать»)

| Задача | Рекомендация |
|--------|--------------|
| **Код (Python/JS)** | Claude Sonnet / GPT-4 — лучшее качество кода |
| **Большой контекст (>100k)** | Kimi / Minimax |
| **Аналитический разбор на русском** | Qwen (хорош для русского) |
| **Быстрые ответы** | DeepSeek V3 (быстрый) |
| **Логика / математика** | DeepSeek R1 (Глубокое мышление) |
| **Поиск актуальной информации** | Perplexity |
| **Продакшн в n8n (бюджет)** | YandexGPT |

### Чего НЕ делать
- Не использовать одну модель для всех задач.
- Не использовать модель без few-shot примеров для структурных задач.
- Не доверять модели без надсмотрщика, если задача критичная (Метавр — яркий пример).
