## 6 концепций (по версии Qwen, плюс проверка)

### 1. Context Engineering ✅ подтверждено
**Фокус смещается с промптов на проектирование информационной среды**: что положить в контекст, в каком формате, с какими примерами.

> Подтверждено в сообществе как реальная концепция Karpathy.

### 2. LLM как усилитель чтения ✅ подтверждено
**Три слоя анализа документа:**
1. **Skim** — беглый просмотр.
2. **Summary** — сжатие тезисов.
3. **Meta-analysis** — попросить LLM указать, какие идеи были бы **новыми лично для тебя**.

### 4. LLM как симулятор реальности
LLM не имеет мнения, это движок для генерации аргументов. Используем слабость (угодничество):
1. Пишем свою позицию.
2. В **новом** чате просим LLM сформулировать **контраргументы**.
3. Сравниваем.

### 5. Markdown-база знаний (LLM Wiki) ✅✅ верифицировано по первоисточникам

> **«3 апреля 2026 Karpathy опубликовал Gist с описанием персональной системы знаний, которую он называет LLM Wiki — трёхуровневая markdown-архитектура».**
> (источник: atlan.com, juejin.cn, news.qq.com, blog.csdn.net, medium.com)

**Архитектура (3 уровня):**
```
raw/      ← исходные материалы (только для чтения)
wiki/     ← структурированные .md файлы (создаёт LLM)
schema/   ← правила оформления (CLAUDE.md / AGENTS.md)
```

**Операционный цикл (от Karpathy):**
1. **Ingest** — LLM читает raw/, анализирует, классифицирует
2. **Compile** — генерирует структурированные .md с обратными ссылками
3. **Query** — вопросы к wiki (это и есть Karpathy's point — не RAG)
4. **Lint** — периодическая проверка (противоречия, устаревшее)

**Ключевая метафора:** RAG — это интерпретатор (каждый раз заново); LLM Wiki — это **компилятор** (один раз скомпилировал, дальше используем результат).

**«Compounding effect» (сложный процент):** с каждым новым источником wiki становится всё полезнее, потому что новые знания ссылаются на старые.

**Инструмент Karpathy:** Obsidian как «фронтенд IDE» для wiki (просмотр, ручная правка, поиск).

### 6. Практические советы промптинга
- **Не «как ты считаешь?»**, а «какие позиции заняли бы разные эксперты?».
- **Держать ИИ на поводке:** маленькие шаги, вы — узкое горлышко.

## 10 железных принципов (из чата про «совет экспертов»)

| № | Принцип | Суть |
|---|---------|------|
| 1 | Формулируй как задачу | «Сделай X», не «как думаешь?» |
| 2 | Требуй рассуждения (CoT) | Сначала покажи ход мыслей |
| 3 | Few-shot примеры | 2-3 примера > длинные объяснения |
| 4 | Принудительная самооценка | Пусть LLM сама себя критикует |
| 5 | Бинарные оценки | Принято/не принято, не «1-5» |
| 6 | Один запрос = один файл | Не перегружай контекст |
| 7 | Разделяй роли LLM | Perplexity для поиска, Kimi для текста, и т.д. |
| 8 | Условие остановки | «5 попыток, потом "не удалось"» |
| 9 | System prompt = конституция | Задавай жёсткие рамки |
| 10 | Чёрный список приёмов | Записывай что НЕ работает |

## Новые концепции (по сообществу)

- **Agentic Engineering** (Karpathy, 2026) — новая парадигма поверх «vibe coding». Ставишь задачу, границы, критерии качества — агент реализует.
- **RLVR** (Reinforcement Learning from Verifiable Rewards) — прорыв 2025.
- **«Jagged intelligence»** — интеллект LLM фрагментирован: в одной задаче гений, в соседней — нет.

## Сравнение LLM-клиентов (от Qwen, из чата)

| Сервис | Главная сила | Идеальный сценарий |
|--------|--------------|---------------------|
| **DeepSeek** | Глубокий анализ, логика, длинный контекст (128k-1M) | Разбор сложных концепций, исследование |
| **Kimi** | Сверхдлинный контекст (2 млн символов) | Чтение и анализ больших текстов |
| **Qwen** | Универсальность + мультимедиа | Текст + картинки, быстрые прототипы |
| **Minimax** | Экстремальный контекст (4 млн токенов) | Длинные проекты, целые книги |
| **Perplexity** | Актуальный поиск + синтез | Фактчекинг, быстрые обзоры |
