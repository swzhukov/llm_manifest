# Сергей как тестер LLM-моделей

> **Источники:** `raw_imports/00_DS_Лучшая модель выбора.md` (2026-06-15), `raw_imports/01_DS_Дипсик и его трепыхания.md` (2026-06-09), `raw_imports/03_DS_Помощь с решением задачи.md` (2026-06-08), `raw_imports/35_QW_Анализ ответов моделей и улучшение скрипта.md` (2026-05-27).

> Сергей — **активный тестер LLM-моделей**. Он не просто использует модели, а **сравнивает их на реальных задачах** и **фиксирует**, какая лучше.

---

## Паттерн поведения

1. Берёт **конкретную задачу** из своей работы (код, анализ, план).
2. Даёт **одну и ту же задачу** нескольким моделям (DeepSeek, Kimi, Minimax, Qwen, Perplexity).
3. **Сравнивает результаты** по качеству, скорости, цене.
4. Выбирает лучшую для этой задачи.
5. **Документирует** выводы в чатах.

---

## Какие модели тестирует (по факту в чатах)

| Модель | Где тестировал |
|--------|----------------|
| **DeepSeek** (V3, V3.1, R1) | Основная. Сравнивает с Kimi, Qwen. Часто разочарован. |
| **Kimi** | Сравнивает с DeepSeek. Сложно оценить из чатов — то ли нравится, то ли нет. |
| **Qwen** | Использует как «альтернативу» DeepSeek. |
| **Minimax** | Упоминается в Karpathy-чате как «тяжёлая артиллерия» для больших объёмов. |
| **Perplexity** | Для поиска актуальной информации. |
| **YandexGPT** | В n8n-агентах (продакшн). |

## Какие задачи тестирует

### Задача 1: Генерация длинного Python-кода
- **YouTube → Google Sheets** (Qwen, 48 сообщений — самая длинная итерация)
- **Сравнение:** DeepSeek (жалкие трепыхания) vs Kimi vs Qwen
- **Вывод:** Qwen лучше (длинный, рабочий код)

### Задача 2: Аналитический разбор
- **Кадровые проблемы компании** (Qwen, 32 сообщения)
- **Сравнение:** DeepSeek vs Qwen
- **Вывод:** Qwen даёт более глубокий разбор

### Задача 3: Аудит встречи (B2B продажи)
- **Несколько чатов** (QW 19, 21, 22, 23, 24, 25 — 6+ продающих встреч)
- **Сравнение:** Qwen справляется лучше DeepSeek для этой задачи
- **Вывод:** Qwen — основной для продаж

### Задача 4: Генерация ТЗ
- **Инвестиционный агент** (DS 05, 07)
- **Сравнение:** DeepSeek даёт хорошее ТЗ, но требует уточнений
- **Вывод:** DeepSeek ОК для ТЗ, но не для творческих задач

### Задача 5: Научный разбор
- **Книга о долголетии** (DS 09)
- **Сравнение:** DeepSeek даёт пересказ вместо критики
- **Вывод:** DeepSeek **не справился** — Сергей остался разочарован

### Задача 6: Помощь с задачей (поиск решения)
- **«Помощь с решением задачи»** (DS 03) — длинный код
- **Сравнение:** Qwen vs Kimi vs DeepSeek
- **Вывод:** Сергей сам переделывал код после DeepSeek

## Параметры выбора (что для Сергея важно)

Из чатов:
- **Точность** — главный приоритет.
- **Учёт контекста** — модель должна понимать специфику задачи.
- **«Не задавай лишних вопросов»** — модель должна использовать данные.
- **Длина контекста** — для больших задач (транскрипты, ТЗ).
- **«Глубокое мышление» (R1)** — для логических задач.

**Что не важно (или менее важно):**
- Скорость — нет жёстких требований.
- Цена — для MVP-этапа не приоритет.

| Чат | Что произошло | Модель |
|-----|---------------|--------|
| 00_DS | Сергей спрашивает «какую модель выбрать» | DeepSeek |
| 01_DS | «Жалкие трепыхания дипсика» | DeepSeek |
| 03_DS | Переделывает код за DeepSeek | DeepSeek |
| 12_DS | 3 раза пнул — не оценил Метавр | DeepSeek |
| 15_DS | «Ты сам ничего не нашёл?» | DeepSeek |
| 09_DS | «Научный разбор, не пересказ» | DeepSeek |

**Вывод:** Сергей **не доверяет DeepSeek для творческих задач**, но продолжает использовать (видимо, из-за стоимости или доступности).

**Qwen** — единственная модель, которой он регулярно доверяет для серьёзных задач.

## Рекомендации для меня (Mavis)

1. **Не быть DeepSeek** — не задавать вопросов, использовать данные.
2. **Структурировать вывод** (по умолчанию).
3. **Критиковать себя** — предлагать анти-паттерны и улучшения.
4. **Few-shot примеры** — при структурных задачах.
5. **Длинный контекст** — у меня есть, использовать на полную.
6. **Не уклоняться** — если Сергей дал задачу с данными, использовать их все.
