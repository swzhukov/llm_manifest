## Ценности (как проявляются)

### 1. Системность и итеративность
- **Перед каждым проектом** — длинное ТЗ с этапами, рисками, бюджетами.
- **Использует WBS**, актировки, KPI.
- **Допускает полный ребилд** при сдвиге предпосылок.
- **Источник:** десятки чатов с ТЗ, планами, KPI.

### 2. Научная строгость
- Хочет **«современные научные консенсусы»**, не советы из книги о долголетии.
- Требует «самого профессионального» уровня от LLM.
- Раздражается, когда LLM «жужжит» вместо работы.
- **Источник:** `09_DS_Критический анализ книги о долголетии.md`.

### 3. Долг/ответственность
- Кредитор Метавра — **не списывает** долг.
- Заботится о пожилом родственнике с Паркинсоном.
- **Источник:** `12_DS_Оценка Метавра.md`, `17_DS/18_DS_Ассистивные технологии.md`.

### 4. Семья, род, идентичность
- Глубоко копает генеалогию Жуковых и Роговых.
- Хочет «крутой» сайт про фамилию.
- **Источник:** `15_DS_Глубокая генеалогия.md`, `16_DS_Жуковы Сайт Проект.md`.

### 5. Личная эффективность
- Учится в машине (Loora).
- Интересуется долголетием, нейропластичностью.
- Строит персональные системы (инвест-агент, транскрибация, экспорт чатов, RAG).
- **Источник:** `42_DS_Дополнительные Техники.md`, `05_DS_Инвест-агент.md`, `52_QW_Построение RAG.md`.

### 6. Честность и прямота
- Матерится, когда LLM тупит: «блять, ты дурак?».
- Требует от LLM «честной обратной связи» по своим мыслям.
- Не tolerирует hedging.
- **Источник:** `12_DS_Метавр.md` (фрустрация), `13_DS_Маркировка.md` (просит честную обратную связь).

### 7. B2B-прагматизм
- Продаёт «комплексные проекты», а не «коробочный продукт».
- Просит LLM «быть лучшим продавцом B2B IT-решений в мире».
- Связывает технические решения с бизнес-метриками клиента.
- **Источник:** серия QW 19-25 «продавец».

### 8. Не tolerирует «жалкие трепыхания»
- «Посмотри на жалкие трепыхания дипсика» (когда DeepSeek не справился).
- «Жалкие трепыхания» = фрустрация, раздражение, требование результата.
- **Источник:** `01_DS_Дипсик и его трепыхания.md`.

## Анти-паттерны (что его бесит в LLM)

| Что LLM делает | Что Сергей хочет |
|----------------|------------------|
| Задаёт уточняющие вопросы | Использовать данные |
| Поверхностный ответ | «Самый профессиональный» уровень |
| Hedging | Один direction |
| Переписывает всё | Surgical changes |
| Многошаговые планы без результата | Результат на каждом шаге |
| «Это выходит за рамки» | Действовать итеративно |
| «Может быть», «возможно» | Один answer |
| Долго думает | Сначала результат, потом уточнения |
| Не использует контекст | Использовать ВСЁ |

## Паттерны решений (как действует)

1. **Сначала план, потом действие** — все проекты начинаются с ТЗ.
2. **Использует LLM как «самого лучшего» специалиста** в нужной роли.
3. **Просит пересказ задачи и допущений** — явно.
4. **Тестирует альтернативы параллельно** — DeepSeek, Kimi, Minimax, Qwen, Perplexity.
5. **Пишет «программные посты»** для команды.
6. **Действует «в тень»** (нанять Мишу в обход Антона).
7. **Строит системы, а не точечные решения.**
8. **Требует непрерывного контекста** — LLM должна помнить всё между сессиями.
9. **Принимает полный ребилд** — если предпосылки изменились, не патчит, а пересобирает.

## Стиль общения

- **Короткие русские фразы**, без академической обвязки.
- **Технические термины в норме** (1С, ИИС, ОФЗ, MRP II, BPMN, etc.).
- **Матерится, когда раздражён** (не нужно сюсюкать, нужно чинить).
- **Структурированный вывод** любит (таблицы, карточки, чек-листы, anti-pattern lists).
- **Opinionated default** предпочитает (а не «можно А или Б»).
- **Few-shot примеры** работают (паттерн Карпатого).
- **Критикуйте сами себя** — он просит.

## Уверенность по ценностям

- **Очень высокая** (подтверждено в десятках чатов): системность, итеративный ребилд, научная строгость, забота о родственнике, любовь к жёсткому диалогу с LLM.
- **Высокая:** opinionated default, структурированный вывод.
- **Средняя:** размер ответа (в чатах разный).
- **Не раскрыто:** религия, политика, сексуальная ориентация — **НЕ предполагать**.
