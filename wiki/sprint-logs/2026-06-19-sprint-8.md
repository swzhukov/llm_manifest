### 2. Найдены расхождения

| Расхождение | Где | Что в реализации | Что должно быть |
|---|---|---|---|
| **user_profile** | `Code — Build YandexGPT payload` (n8n) | Хардкод placeholder | Реальный профиль из KB |
| **prompts** | Тот же | Общая фраза "долгосрочный портфельный инвестор" | Конкретные инструменты, watchlist, ИИС, налоги |

### 3. Реализованные фиксы

#### 3a. v6.0.5: `vtt_to_claims()` в `utils.py`
Парсит VTT в список `[{ts_in_video: float, text: str}]`. Dedup identical.
1272 cues → 641 unique claim.

#### 3b. v6.0.5: `/youtube_subs` возвращает `claims + timeline`
- `claims` — массив с ts (для UI)
- `timeline` — строковый формат `[hh:mm:ss] text` (для YandexGPT input)

#### 3c. v6.0.5/6.0.6: server-side `_build_investor_prompt()` в `/yagpt_summarize`
Принимает `user_profile` dict, генерирует жёсткий JSON prompt с:
- ПРОФИЛЬ (горизонт, цель, инструменты, watchlist, риск, любимые блогеры)
- ⚠️ КРИТИЧНО: даже если видео не про инвестиции, actions ОБЯЗАНЫ ссылаться на watchlist/ОФЗ/ИИС/налоги
- Confidence 0.8-1.0 для инвестиционных видео, 0.4-0.6 для непрофильных
- ПРИМЕРЫ правильных actions (X5, MGN, LSR, OFZ-26248)

#### 3d. v6.0.7: smart claims rendering в `_render_html`
- Merge consecutive cues within 3 sec gap в одну фразу
- Filter `len(text) >= 40` chars (real phrases, not fragments)
- Dedupe by normalized text (lowercase, strip punctuation)
- Format ts as `mm:ss` или `hh:mm:ss`
- Top 30 by ts order

#### 3e. v6.0.8: cumulative overlap strip в `vtt_to_claims`
- Strip prefix-suffix overlap между consecutive cues
- Sliding window last 200 chars для next overlap check
- NoneType guard

### 5. Cost

- Slightly higher than v6.0.1 (0.96₽) из-за расширенного prompt с user_profile — но trade-off оправдан (relevance × 10)

### 6. Lessons learned (MISTAKES §3.30-3.33)

- **§3.30** Hardcoded user_profile в n8n Code-ноде → server-side builder в Flask
- **§3.31** VTT auto-transcript cumulative cues → strip overlap (sliding window 200 chars)
- **§3.32** NoneType guard при `prev_text + ' ' + text` — паттерн с `if X: X = X + Y` всегда требует `else: X = Y`
- **§3.33** `FLASK_DEBUG=0` всё равно кэширует модули — `rm -rf __pycache__` обязателен при hot-patch
