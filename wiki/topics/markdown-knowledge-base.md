# Концепция 5: Markdown-базы знаний (LLM Wiki)

**Автор:** Андрей Карпатый
**Проблема:** LLM теряют контекст в длинных диалогах.
**Решение:** LLM сама ведёт Markdown-вики, компилируя знания.

## Структура
- `raw/` – неизменяемые исходники.
- `wiki/` – скомпилированные страницы.
- `SCHEMA.md`, `AGENTS.md` – правила для LLM.

## Операционный цикл
1. **Ingest** – добавить источник в `raw/`, LLM обновляет `wiki/`.
2. **Query** – задать вопрос на основе вики.
3. **Lint** – периодическая проверка целостности.

## Преимущества
- Прозрачность (человек читает `.md` файлы).
- Знания накапливаются.
- Можно версионировать.
