# Манифест Совета экспертов: 10 железных принципов

## 10 железных принципов
| № | Принцип | Суть |
|---|---------|------|
| 1 | **Формулируй как задачу** | LLM лучше решает, когда чётко сказано, что сделать. |
| 2 | **Требуй рассуждения (CoT)** | Заставляй модель "думать вслух". |
| 3 | **Давай 2-3 примера (few-shot)** | Примеры работают лучше объяснений. |
| 4 | **Принудительная самооценка** | После ответа попроси LLM найти слабые места. |
| 5 | **Бинарные оценки** | Используйте "принято/не принято" для каждого выхода. |
| 6 | **Один запрос – один файл** | Не перегружайте контекст. |
| 7 | **Разделяй роли между LLM** | У каждого сервиса есть суперсила. |
| 8 | **Условие остановки** | Задавай лимит итераций для многошаговых задач. |
| 9 | **Системный промпт как конституция** | Задавай жёсткие рамки первым сообщением. |
| 10 | **Чёрный список приёмов** | Избегай многозадачности в одном сообщении. |

## 🧩 Пять сценариев с готовыми промптами

### Сценарий 1: Анализ сложной статьи (Kimi/Minimax)
~~~
Ты – профессиональный аналитик текста. Выполни следующие шаги:
1. Разбей текст на логические разделы.
2. Для каждого раздела выдели одну главную мысль и 1-2 ключевых факта.
3. Составь общую сводку из 5-7 пунктов.
4. Укажи спорные утверждения.
Формат вывода: Markdown.
~~~

### Сценарий 2: Сложная логическая/кодовая задача (DeepSeek R1)
~~~
Перед тем как дать ответ:
1. Переформулируй задачу своими словами.
2. Перечисли все допущения.
3. Разработай два подхода, сравни их.
4. Выбери лучший и реализуй.
5. Напиши тест, доказывающий работу решения.
Задача: [опишите здесь]
~~~

### Сценарий 3: Проверка решения на слабые места (Perplexity + DeepSeek)
**Шаг 1 (Perplexity):** Найди три критических замечания к подходу [X]. Дай ссылки.
**Шаг 2 (DeepSeek):** Вот мой подход: [текст]. Вот аргументы против: [вставить]. Напиши эссе из трёх абзацев: сильные стороны, слабые с учётом аргументов, предложения по улучшению.

### Сценарий 4: Структурирование заметок (Qwen)
~~~
Преврати сырые заметки в структурированный документ:
- Создай оглавление.
- Сгруппируй идеи по темам.
- Для каждой темы – один абзац + ключевые термины (жирным).
Заметки: [вставить]
~~~

### Сценарий 5: Спасение контекста (любой клиент)
~~~
Игнорируй историю, кроме этого сообщения. Напиши сжатое саммари диалога:
- Исходная задача.
- Все принятые решения.
- Открытые вопросы.
- Три важнейших вывода.
Формат: bullet points, не более 15 пунктов.
~~~

## 🔧 Особенности веб-клиентов
| Сервис | Когда использовать | Что нельзя делать |
|--------|--------------------|-------------------|
| **DeepSeek** | Логика, код, «подумать вслух». | Загружать огромные файлы. |
| **Kimi** | Анализ документов >100 страниц. | Просить быстрые короткие ответы. |
| **Minimax** | Мега-объёмы (до 4 млн токенов). | Для простых одноразовых вопросов. |
| **Qwen** | Мультимодальность: картинки, схемы. | Ожидать глубокой логики уровня R1. |
| **Perplexity** | Поиск актуальных данных, фактчекинг. | Использовать без интернета. |
