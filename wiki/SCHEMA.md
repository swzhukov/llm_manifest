# Схема базы знаний (LLM Wiki)

## Структура папок
- `wiki/` – корень базы знаний
  - `SCHEMA.md` – этот файл
  - `AGENTS.md` – правила поведения LLM (4 принципа Карпатого)
  - `README.md` – оглавление
  - `index.md` – полный индекс с аннотациями и тегами
  - `glossary.md` – глоссарий терминов
  - `session-summaries.md` – хронология диалога
  - `pending.md` – отложенные вопросы
  - `log.md` – журнал изменений
  - `COUNCIL_RECOMMENDATIONS.md` – манифест Совета экспертов
  - `raw/` – исходные материалы (не редактируются LLM)
  - `topics/` – детальные страницы по темам
  - `templates/` – шаблоны для новых заметок

## Правила ведения
1. **Frontmatter:** каждый файл начинается с YAML-блока (title, tags, created).
2. **Обновление:** после значимого сообщения LLM обновляет файлы (добавляет, не перезаписывает без версии).
3. **Версионирование:** пометка `[Updated: YYYY-MM-DD]` или `status: superseded`.
4. **Ссылки:** относительные, в стиле `[термин](./glossary.md#термин)`.
5. **Аудит:** команда `lint wiki` запускает проверку целостности.

## Обязательные файлы
- `README.md`, `index.md`, `glossary.md`, `session-summaries.md`, `pending.md`, `log.md`, `SCHEMA.md`, `AGENTS.md`, `raw/sources.md`
- Папка `topics/` должна содержать минимум 5 файлов.
