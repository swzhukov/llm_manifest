# Глоссарий

## A–C
- **AGENTS.md** – файл с правилами для LLM-агентов.
- **Agentic Engineering** – парадигма, пришедшая на смену Vibe Coding (2026).
- **Context Engineering** – проектирование информационной среды (контекстного окна).
- **Context Rot** – затухание релевантности старых данных в длинном диалоге.
- **Chain-of-Thought (CoT)** – техника принудительного рассуждения.
- **Critique Shadowing** – метод оценки LLM через комментарии эксперта.

## K–M
- **LLM Wiki** – концепция Карпатого №5: база знаний в Markdown, которую LLM ведёт сама.
- **Loop Engineering** – создание автоматических циклов обратной связи (Maker–Checker).
- **Maker–Checker** – паттерн: один агент генерирует, другой проверяет.

## P–S
- **Pending** – раздел вики для отложенных вопросов.
- **RLVR** – метод обучения LLM с проверяемым вознаграждением.
- **Surgical changes** – принцип изменения только того, что относится к задаче.
- **System prompt** – инструкция, задающая LLM правила на весь диалог.
- **Vibe Coding** – ранняя парадигма (2025) с нечёткими запросами.
